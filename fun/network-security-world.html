Thế Giới Network Security

Từ những ngày đầu Internet hoang dã đến kiến trúc Zero Trust hiện đại. Mọi thứ bạn cần biết về bảo mật mạng - các loại tấn công, lớp phòng thủ, công cụ phân tích, và cách xây dựng hệ thống an toàn ngay trong homelab của bạn.

⌄

Network Security là gì thực sự?

Network Security - không đơn giản là "cài firewall rồi xong". Đó là một hệ thống chiến lược nhiều tầng, nhiều lớp, được thiết kế để bảo vệ tính toàn vẹn, tính sẵn sàng, và tính bảo mật của dữ liệu khi nó di chuyển qua mạng. Hãy tưởng tượng mạng của bạn như một thành trì thời trung cổ: không chỉ có tường thành, mà còn hào nước, cổng thành, lính gác, và cả hệ thống cảnh báo sớm.

Triết lý cốt lõi là Defense in Depth - phòng thủ theo chiều sâu. Không một lớp bảo vệ nào là đủ. Khi lớp ngoài bị phá, lớp trong phải sẵn sàng. Khi firewall bị vượt qua, IDS phải phát hiện. Khi IDS bỏ sót, SIEM phải tương quan log và cảnh báo. Đó là nghệ thuật xếp chồng các lớp phòng thủ để kẻ tấn công phải vượt qua tất cả - trong khi bạn chỉ cần phát hiện chúng ở bất kỳ lớp nào.

Bộ ba CIA Triad - Nền tảng mọi quyết định bảo mật

🔒

Confidentiality

Tính bảo mật

Chỉ người được phép mới đọc được dữ liệu. Encryption, access control, classification. Nếu ai đó nghe lén được traffic của bạn nhưng chỉ thấy dữ liệu đã mã hoá - confidentiality vẫn được bảo toàn.

✅

Integrity

Tính toàn vẹn

Dữ liệu không bị sửa đổi trái phép trong quá trình truyền. Hashing, checksums, digital signatures. Bạn gửi "chuyển 100$" nhưng kẻ tấn công sửa thành "chuyển 10,000$" - đó là vi phạm integrity.

🌐

Availability

Tính sẵn sàng

Hệ thống phải hoạt động khi cần. DDoS protection, redundancy, failover. Server bạn bảo mật tuyệt đối nhưng bị đánh sập - availability = 0. Bảo mật tốt nhất là vô nghĩa nếu không ai dùng được.

Insight quan trọng

Ba yếu tố CIA thường mâu thuẫn nhau. Mã hoá mạnh (Confidentiality) làm chậm hệ thống (giảm Availability). Cho phép truy cập rộng (Availability) làm giảm bảo mật (Confidentiality). Nghệ thuật của security engineer là cân bằng ba yếu tố này phù hợp với ngữ cảnh cụ thể của tổ chức.

Dòng Thời Gian Network Security

Từ Internet hoang dã không mật khẩu đến thế giới Zero Trust - cuộc chạy đua vũ trang không hồi kết

1969-1988

Internet nguyên thuỷ - Thời kỳ tin tưởng mù quáng

ARPANET ra đời trong môi trường học thuật, nơi mọi người đều tin tưởng nhau. Không có khái niệm authentication hay encryption. Mật khẩu lưu dạng plaintext. Telnet, FTP gửi credential không mã hoá. Một thế giới đẹp đẽ nhưng ngây thơ đến nguy hiểm.

1988

Morris Worm - Tiếng chuông cảnh tỉnh

Robert Tappan Morris, nghiên cứu sinh Cornell, phát tán worm đầu tiên trên Internet. Lây nhiễm khoảng 6,000 máy (10% Internet thời đó). Thiệt hại ước tính 10-100 triệu USD. Hậu quả: CERT/CC (Computer Emergency Response Team) được thành lập. Internet nhận ra rằng nó cần bảo mật.

1990s

Kỷ nguyên Firewall

Packet filter firewall ra đời (stateless, chỉ nhìn header). Rồi stateful firewall (theo dõi connection state). Check Point, Cisco PIX thống trị thị trường. Triết lý "xây tường thành, chặn kẻ xấu bên ngoài" - castle-and-moat model. Đơn giản nhưng hiệu quả cho thời đại đó.

1998-2005

IDS/IPS - Từ phát hiện đến ngăn chặn

Snort ra đời (1998) - IDS open source huyền thoại. Martin Roesch tạo ra quy tắc mới: không chỉ chặn ở biên, mà còn phải giám sát traffic bên trong. Từ IDS (chỉ cảnh báo) tiến hoá thành IPS (tự động block). Network-based và Host-based IDS song song phát triển.

2005-2015

SIEM - Tương quan và phân tích

Security Information and Event Management. Thu thập log từ mọi nguồn, tương quan sự kiện, phát hiện pattern tấn công phức tạp. Splunk, ArcSight, QRadar. Lần đầu tiên, security team có "bức tranh toàn cảnh" thay vì nhìn từng mảnh rời rạc. Nhưng cũng sinh ra vấn đề mới: alert fatigue.

2014-2020

Zero Trust - Tin ai cũng chết

Google công bố BeyondCorp paper (2014) sau vụ Operation Aurora. Triết lý mới: "Never trust, always verify". Không có khái niệm "mạng nội bộ an toàn". Mọi request đều phải xác thực, bất kể từ đâu. Identity-based access thay thế network-based access. Mô hình castle-and-moat chính thức sụp đổ.

2020-nay

AI-Driven Security & Tự động hoá

Machine Learning phát hiện anomaly mà rule-based không thể. SOAR (Security Orchestration, Automation and Response) tự động hoá incident response. XDR mở rộng phạm vi phát hiện. Cuộc chiến giữa AI phòng thủ và AI tấn công bắt đầu. Deepfake, AI-generated phishing - kẻ tấn công cũng dùng AI.

DDoS

Distributed Denial of Service

Hàng triệu request đổ về cùng lúc, nhấn chìm server. Có nhiều dạng: volumetric (UDP flood, amplification), protocol (SYN flood, Ping of Death), application layer (HTTP flood, Slowloris). Botnet với hàng trăm nghìn thiết bị IoT bị chiếm quyền trở thành vũ khí. Mirai botnet (2016) từng đánh sập Dyn DNS, khiến nửa Internet Mỹ gián đoạn.

Man-in-the-Middle (MITM)

Nghe lén & can thiệp

Kẻ tấn công chèn mình vào giữa hai bên giao tiếp. Đọc, sửa, hoặc tiêm dữ liệu mà cả hai bên không hay biết. WiFi công cộng là môi trường lý tưởng. SSL stripping biến HTTPS thành HTTP. Trên mạng LAN, kết hợp ARP spoofing + MITM proxy là combo kinh điển. Ngay cả HTTPS cũng bị bypass nếu nạn nhân chấp nhận certificate giả.

ARP Spoofing

Đầu độc bảng ARP

Gửi ARP reply giả mạo, nói với switch/host rằng "MAC address của gateway là MAC của tôi". Tất cả traffic trong LAN bắt đầu chảy qua máy kẻ tấn công. Là bước đệm cho MITM, sniffing, session hijacking. Cực kỳ hiệu quả trên mạng LAN không có bảo vệ. Phòng chống: Dynamic ARP Inspection (DAI) trên switch, static ARP entries, VLAN segmentation.

Layer 2 LAN only Bước đệm MITM DAI phòng chống
🧪

DNS Poisoning

Đầu độc DNS Cache

Chèn record DNS giả vào cache của resolver. Nạn nhân gõ "facebook.com" nhưng bị dẫn đến server giả mạo. Dan Kaminsky (2008) phát hiện lỗ hổng DNS nghiêm trọng ảnh hưởng toàn bộ Internet. Phòng chống: DNSSEC (ký số DNS response), DNS-over-HTTPS (DoH), DNS-over-TLS (DoT). Pi-hole và AdGuard giúp kiểm soát DNS tại nhà.

Integrity Attack DNS Layer DNSSEC DoH/DoT
🔍

Port Scanning

Reconnaissance / Dò quét

Bước đầu tiên của mọi cuộc tấn công: tìm hiểu mục tiêu có gì đang mở. SYN scan (half-open, nhanh, ít bị log), connect scan, UDP scan, FIN/NULL/XMAS scan để bypass firewall. Nmap là vua của port scanning. Một port mở không mong muốn = một cánh cửa cho kẻ tấn công. Mỗi service chạy trên port đó là một bề mặt tấn công tiềm năng.

Recon phase Nmap SYN/TCP/UDP

Brute Force & Credential Stuffing

Thử mật khẩu hàng loạt

Thử tất cả tổ hợp mật khẩu có thể. Dictionary attack dùng wordlist phổ biến (rockyou.txt có 14 triệu password). Credential stuffing dùng password bị leak từ vụ breach khác. Hydra, Medusa, John the Ripper là công cụ kinh điển. SSH, RDP, web login là mục tiêu hàng đầu. Rate limiting và fail2ban là phòng thủ cơ bản nhất.

SQL Injection

Tiêm mã SQL

Chèn mã SQL vào input của ứng dụng web. Câu kinh điển: ' OR '1'='1. Kẻ tấn công có thể đọc toàn bộ database, xoá dữ liệu, thậm chí chiếm quyền server. OWASP Top 10 luôn có mặt SQLi. Phòng chống: prepared statements, parameterized queries, WAF. Không bao giờ trust user input.

Cross-Site Scripting (XSS)

Tiêm mã JavaScript

Chèn JavaScript độc hại vào trang web mà người dùng khác sẽ xem. Stored XSS (lưu trong DB), Reflected XSS (qua URL), DOM-based XSS. Đánh cắp cookie session, keylogging, redirect đến trang giả. BeEF framework biến browser nạn nhân thành zombie. Content Security Policy (CSP) và output encoding là phòng thủ chính.

Phishing & Social Engineering

Lừa đảo qua tâm lý

Yếu tố con người là mắt xích yếu nhất. Email giả mạo ngân hàng, trang đăng nhập clone hoàn hảo, cuộc gọi từ "bộ phận IT". Spear phishing nhắm vào cá nhân cụ thể, whaling nhắm vào lãnh đạo. Với AI tạo nội dung, phishing ngày càng tinh vi. 91% cyberattack bắt đầu bằng phishing email. Security awareness training là vaccine tốt nhất.

IDS / IPS

Intrusion Detection & Prevention

Suricata - đa luồng, hiệu năng cao, hỗ trợ Lua scripting, đang thay thế Snort. Snort - OG của IDS, hàng triệu rule từ cộng đồng, Cisco mua và phát triển Snort 3. WAF

Web Application Firewall

Chuyên bảo vệ ứng dụng web. Chặn SQLi, XSS, CSRF, path traversal ở tầng HTTP. ModSecurity (open source, tích hợp Nginx/Apache) với OWASP Core Rule Set. Cloudflare WAF cho cloud. Khác firewall thông thường: WAF hiểu HTTP request/response, có thể phân tích body, cookie, header. Virtual patching khi chưa kịp sửa code.

📊

SIEM

Security Information & Event Management

Bộ não trung tâm. Thu thập log từ firewall, IDS, server, endpoint, application. Tương quan sự kiện từ nhiều nguồn để phát hiện tấn công phức tạp mà từng thành phần riêng lẻ không thể thấy. Ví dụ: login thất bại trên firewall + port scan + new process trên server = đang bị tấn công. Chi tiết hơn ở phần SIEM riêng bên dưới.

Correlation Log analysis Cần tuning
🖥️

EDR

Endpoint Detection & Response

Cài agent trên mỗi endpoint (PC, server, laptop). Giám sát process, file, registry, network connection realtime. Phát hiện malware, lateral movement, fileless attack. Khác antivirus: EDR không chỉ quét signature mà phân tích hành vi (behavioral analysis). CrowdStrike Falcon, SentinelOne, Wazuh agent là ví dụ. Đặc biệt quan trọng trong thời đại WFH.

Behavioral Agent-based Wazuh CrowdStrike

NAC (802.1X)

Network Access Control

Kiểm soát ai được phép kết nối vào mạng. 802.1X xác thực thiết bị trước khi cho phép truy cập port. RADIUS server (FreeRADIUS) làm backend authentication. Thiết bị không xác thực được = đẩy vào VLAN cách ly hoặc bị block hoàn toàn. Kết hợp certificate-based auth cho zero-touch onboarding. Ngăn chặn rogue device ngay từ tầng physical.

Nguyên tắc vàng

Không công cụ nào là viên đạn bạc. Firewall không chặn được insider threat. IDS không phát hiện được zero-day nếu chưa có signature. SIEM chỉ tốt bằng rule và log bạn đưa vào. Defense in Depth có nghĩa là chấp nhận mỗi lớp đều có thể thất bại - và thiết kế hệ thống sao cho thất bại ở một lớp không dẫn đến sụp đổ toàn bộ.

Phân Vùng Mạng (Network Segmentation)

Chia để trị - kẻ tấn công phá được một vùng không có nghĩa chiếm được toàn bộ

🏷️

VLANs

Virtual LAN - Tầng cơ bản

Chia một switch vật lý thành nhiều broadcast domain logic. VLAN 10 cho management, VLAN 20 cho servers, VLAN 30 cho users, VLAN 99 cho IoT. Traffic giữa VLAN phải đi qua router/firewall - nơi bạn kiểm soát policy. 802.1Q tagging trên trunk port. Là nền tảng của mọi chiến lược segmentation.

Homelab tip

Với managed switch (TP-Link TL-SG108E ~$30), bạn đã có thể tạo VLAN. Kết hợp pfSense/OPNsense làm inter-VLAN routing với firewall rules. Camera IoT vào VLAN riêng, chặn không cho truy cập Internet trừ NVR.

Cơ bản 802.1Q Layer 2
🏰

DMZ

Demilitarized Zone

Vùng đệm giữa Internet và mạng nội bộ. Web server, mail server, DNS server công khai đặt ở đây. Nếu server trong DMZ bị hack, kẻ tấn công vẫn phải vượt qua firewall thứ hai để vào mạng nội bộ. Dual-firewall DMZ (hai firewall khác vendor) là best practice. Single-firewall DMZ (3 interface: outside, DMZ, inside) phổ biến hơn vì đơn giản.

Best practice Public services Dual firewall
🔬

Micro-segmentation

Zero Trust Network

Đưa segmentation xuống cấp độ workload/container. Mỗi VM, mỗi pod trong Kubernetes có firewall policy riêng. Không chỉ "server VLAN" vs "user VLAN" mà "web-server-01 chỉ được nói chuyện với db-server-01 trên port 3306, không gì khác". VMware NSX, Calico (K8s), Cilium. Triển khai khó nhưng blast radius gần bằng 0.

Insight thực tế

Segmentation không chỉ là chuyện kỹ thuật. Nó đòi hỏi bạn hiểu data flow của tổ chức: service nào nói chuyện với service nào, trên port nào, theo hướng nào. Không hiểu data flow mà cắm đầu segment = break application. Luôn bắt đầu bằng monitoring mode (chỉ log, không block) trước khi enforce policy.

Mã Hoá & Xác Thực

Encryption là phòng tuyến cuối - ngay cả khi kẻ tấn công bắt được traffic, chúng cũng chỉ thấy rác

TLS Everywhere

TLS 1.3 là chuẩn. Mọi traffic nên được mã hoá. Không chỉ HTTPS cho web - mà cả database connections (MySQL over TLS), LDAP (LDAPS), email (STARTTLS), syslog (TLS). Let's Encrypt cung cấp certificate miễn phí. Trong homelab, dùng Caddy reverse proxy tự động TLS hoặc Traefik với ACME.

TLS 1.3Let's EncryptCaddy/Traefik

IPsec VPN

Mã hoá site-to-site và remote access. ESP (Encapsulating Security Payload) mã hoá toàn bộ packet. IKEv2 cho key exchange. Transport mode (chỉ payload) vs Tunnel mode (cả header). WireGuard đang thay thế IPsec cho nhiều use case vì đơn giản hơn nhiều - chỉ 4,000 dòng code so với hàng trăm nghìn của IPsec.

IKEv2ESP/AHWireGuard thay thế

WPA3 & Wireless Security

WiFi cần bảo mật riêng. WPA3-Personal dùng SAE (Simultaneous Authentication of Equals) thay thế PSK - chống offline dictionary attack. WPA3-Enterprise dùng 192-bit CNSA suite. Luôn tách WiFi guest ra VLAN riêng. Hidden SSID không phải bảo mật (dễ phát hiện bằng probe request).

WPA3-SAEWPA2 vẫn phổ biếnWEP = chết

802.1X / RADIUS

Xác thực trước khi cho vào mạng. Supplicant (client) -> Authenticator (switch/AP) -> Authentication Server (RADIUS). EAP-TLS (certificate cả hai phía) là an toàn nhất. FreeRADIUS cho homelab/SMB. Microsoft NPS cho Windows shop. Kết hợp VLAN assignment động: xác thực xong mới biết đưa vào VLAN nào.

EAP-TLSFreeRADIUSDynamic VLAN

Certificate-Based Auth

Mạnh hơn password. PKI (Public Key Infrastructure) với CA nội bộ. mTLS (mutual TLS) - cả client lẫn server đều phải chứng minh danh tính bằng certificate. Step-CA hoặc SmallStep cho homelab PKI. ACME protocol tự động issue/renew. Không còn lo password bị brute force.

mTLSPKIStep-CAKhông password

SSH Hardening

SSH key, không password. Ed25519 key (mạnh, ngắn gọn). Disable password authentication. Disable root login. Đổi port (security through obscurity, nhưng giảm noise). AllowUsers/AllowGroups whitelist. Fail2ban hoặc CrowdSec cho rate limiting. SSH certificate auth cho scale lớn (ssh-keygen -s).

Insight: Mã hoá không phải phép màu

Mã hoá bảo vệ dữ liệu in transit (đang truyền) và at rest (đang lưu). Nhưng in use (đang xử lý) thì dữ liệu phải ở dạng plaintext trong RAM. Side-channel attacks, cold boot attacks nhắm vào giai đoạn này. Confidential Computing (Intel SGX, AMD SEV) đang giải quyết vấn đề này nhưng vẫn rất sớm.

Zero Trust Architecture

"Never trust, always verify" - Mô hình bảo mật cho thế giới không còn perimeter

Tại sao cần Zero Trust?

Mô hình truyền thống: "Bên trong firewall = an toàn, bên ngoài = nguy hiểm". Nhưng thực tế: 60% breach bắt nguồn từ insider hoặc lateral movement sau khi xâm nhập thành công. Cloud computing xoá nhoà khái niệm perimeter. Remote work đặt nhân viên ở khắp nơi. VPN cho phép remote user thấy toàn bộ mạng nội bộ - quá rộng. Zero Trust nói: "Không tin ai cả, kể cả thiết bị đang ngồi trong văn phòng."

👤

Verify Identity

Mọi request phải xác thực. MFA bắt buộc. Không chỉ username/password mà còn device posture, location, time-of-day. Continuous authentication - không phải xác thực một lần rồi tin mãi.

🛡️

Least Privilege

Chỉ cấp quyền tối thiểu cần thiết. User chỉ thấy resource họ cần cho công việc. Just-in-time access: quyền chỉ active khi cần, hết thời gian tự thu hồi. Microsegmentation enforce policy.

🔍

Assume Breach

Thiết kế hệ thống như thể kẻ tấn công đã ở bên trong. Encrypt mọi traffic (kể cả nội bộ). Log mọi thứ. Micro-segment mọi workload. Blast radius phải nhỏ nhất có thể. Nếu bị xâm nhập, giới hạn thiệt hại.

BeyondCorp - Zero Trust của Google

1

Bỏ khái niệm mạng nội bộ tin cậy

Google đối xử mạng văn phòng như mạng quán cà phê. Không có "privileged network". Access không phụ thuộc vào bạn đang ở đâu mà bạn là ai và thiết bị của bạn có an toàn không.

2

Device Trust

Mỗi thiết bị có "trust score" dựa trên: OS version, patch level, disk encryption, screen lock. Thiết bị không đạt chuẩn = bị giới hạn access hoặc block hoàn toàn. Device inventory là bắt buộc.

3

Identity-Aware Proxy

Thay vì VPN, sử dụng proxy kiểm tra identity + device trust trước khi cho truy cập application. Mỗi application có policy riêng. User A trên device đạt chuẩn = access. Cùng user A trên device lạ = block.

4

Context-Based Access

Quyết định access dựa trên context đầy đủ: ai (identity), từ đâu (location), thiết bị gì (device trust), truy cập gì (resource), lúc nào (time), có bất thường không (risk score). Không phải binary allow/deny mà là continuous evaluation.

Thực tế cho homelab

Zero Trust đầy đủ cần đầu tư lớn, nhưng bạn có thể áp dụng từng phần: 1) MFA cho mọi service (Authelia, Authentik). 2) WireGuard thay VPN truyền thống, chỉ expose đúng service cần. 3) mTLS giữa các service. 4) VLAN segmentation nghiêm ngặt. 5) Principle of Least Privilege cho mọi user/service account. Đó đã là 80% Zero Trust rồi.

Security Trong Homelab

Từ lý thuyết đến thực hành - hardening checklist cho homelab thực tế của bạn

✅

Hardening Checklist

Danh sách kiểm tra bảo mật
  • SSH: Disable password auth, dùng Ed25519 key, disable root login, đổi port, AllowUsers
  • Firewall: Default deny incoming, chỉ mở port cần thiết, rate limit SSH/HTTP
  • Updates: Unattended-upgrades cho security patches tự động, reboot khi cần
  • Users: Không dùng root, sudo cho từng lệnh, password policy mạnh, MFA khi có thể
  • Network: VLAN segmentation, IoT VLAN riêng, management VLAN riêng
  • TLS: HTTPS cho mọi web service, Let's Encrypt hoặc internal CA
  • DNS: Pi-hole/AdGuard chặn ad & malware domains, DNS-over-TLS upstream
  • Backup: 3-2-1 rule (3 bản sao, 2 loại media, 1 offsite), test restore định kỳ
  • Monitoring: Wazuh agent trên mọi host, log tập trung, alert khi có anomaly
  • Docker: Không chạy container as root, read-only filesystem khi có thể, network isolation
  • Proxmox: Disable public API nếu không cần, 2FA cho web UI, backup schedule
  • Secrets: Không hardcode password trong config file, dùng Vault hoặc ít nhất .env + gitignore

Những Hiểu Nhầm Phổ Biến

8+ myths mà ngay cả người có kinh nghiệm cũng mắc phải

Sai

"Có firewall là đủ an toàn"

Firewall chỉ là một lớp. Nó không chặn được malware trong email attachment, USB drive, hoặc insider threat. Không phát hiện được zero-day exploit. Nếu attacker đã ở bên trong (compromised endpoint), firewall không giúp gì.

Đúng

Firewall là tầng đầu tiên trong chiến lược defense in depth. Cần kết hợp IDS/IPS, EDR, SIEM, network segmentation, và security awareness training. Không bao giờ chỉ dựa vào một công cụ duy nhất.

Sai

"Homelab nhỏ, ai hack làm gì"

Bot tự động không quan tâm bạn là ai. Shodan quét toàn bộ IPv4 mỗi tuần. Một port SSH mở có thể nhận 10,000+ brute force attempts mỗi ngày. IoT devices bị chiếm quyền để gia nhập botnet. Crypto miners tự động tìm server yếu.

Đúng

Mọi thiết bị kết nối Internet đều là mục tiêu. Automated attacks không phân biệt enterprise hay homelab. Bảo mật cơ bản (strong password, firewall, updates) tốn ít effort nhưng chặn được 99% automated attacks.

Sai

"VPN = bảo mật tuyệt đối"

VPN mã hoá traffic giữa bạn và VPN server. Nhưng sau VPN server, traffic lại plaintext (trừ khi đích đến dùng TLS). VPN không bảo vệ khỏi malware, phishing, hoặc compromised website. Và VPN provider có thể log traffic của bạn.

Đúng

VPN hữu ích cho privacy trên mạng không tin cậy (WiFi công cộng) và remote access vào mạng nội bộ. Nhưng không phải giải pháp bảo mật toàn diện. Self-hosted VPN (WireGuard) đáng tin hơn VPN thương mại vì bạn kiểm soát server.

Sai

"Linux không bị hack"

Linux ít malware hơn Windows nhưng KHÔNG miễn nhiễm. Rootkit, kernel exploit, misconfiguration, weak password vẫn là rủi ro. Hầu hết server bị hack chạy Linux. CVE mới cho Linux kernel được phát hiện liên tục.

Đúng

Linux có kiến trúc bảo mật tốt (permission model, SELinux/AppArmor). Nhưng bảo mật phụ thuộc vào cấu hình và bảo trì, không phải OS. Một Linux server không patch, chạy root, password yếu nguy hiểm hơn Windows server được hardening đúng cách.

Sai

"Đổi port SSH là bảo mật"

Đổi port 22 sang 2222 hay 54321 chỉ là security through obscurity. Nmap scan tất cả 65535 port trong vài phút. Banner grabbing cho biết service gì. Giảm log noise nhưng KHÔNG phải biện pháp bảo mật thực sự.

Đúng

Đổi port giảm automated scan noise (tốt cho log). Nhưng bảo mật SSH thực sự: key-based auth, disable password, fail2ban/CrowdSec, AllowUsers, firewall rate limiting. Đổi port là "nice to have", không phải "must have".

Sai

"HTTPS = website an toàn"

HTTPS chỉ đảm bảo traffic được mã hoá giữa browser và server. Phishing site hoàn toàn có thể dùng HTTPS (Let's Encrypt cấp cert cho mọi domain). Website hợp pháp có HTTPS vẫn có thể bị XSS, SQLi, hoặc server bị compromise.

Đúng

HTTPS đảm bảo confidentiality và integrity của kênh truyền. Không nói gì về nội dung hay ý định của website. Luôn kiểm tra domain chính xác, không chỉ biểu tượng ổ khoá. Certificate Transparency logs giúp phát hiện cert giả mạo.

Sai

"Antivirus là đủ cho endpoint security"

Antivirus truyền thống dựa vào signature database - chỉ phát hiện malware đã biết. Fileless malware, living-off-the-land attacks, zero-day không có signature. Polymorphic malware thay đổi hash mỗi lần lây nhiễm. AV bypass là kỹ năng cơ bản của mọi red teamer.

Đúng

Cần EDR (Endpoint Detection and Response) với behavioral analysis - phát hiện hành vi bất thường thay vì so mẫu. Kết hợp application whitelisting, principle of least privilege, và user awareness. AV vẫn hữu ích nhưng chỉ là một lớp.

Sai

"Bảo mật là việc của team IT"

91% cyberattack bắt đầu bằng phishing email nhắm vào nhân viên thường. Social engineering khai thác tâm lý con người, không phải lỗ hổng kỹ thuật. CEO click vào link lạ = toàn bộ hệ thống bị compromise.

Đúng

Bảo mật là trách nhiệm của mọi người. Security awareness training cho toàn bộ tổ chức. Simulated phishing campaigns. Văn hoá "nếu thấy lạ, báo ngay". Kỹ thuật chỉ giải quyết được 50% vấn đề - 50% còn lại là con người.

Sai

"Compliance = Security"

Đạt chuẩn PCI-DSS, ISO 27001, SOC 2 không có nghĩa là bạn an toàn. Compliance là checklist tối thiểu, thường outdated so với threat landscape hiện tại. Nhiều tổ chức bị breach dù đã pass audit.

Đúng

Compliance là baseline, không phải ceiling. Nó đảm bảo bạn không bỏ sót những thứ cơ bản. Nhưng security thực sự đòi hỏi continuous improvement, threat intelligence, incident response practice, và adaptation theo threat mới.

Sai

"Hidden SSID = WiFi an toàn"

Ẩn SSID chỉ khiến AP không broadcast tên mạng. Nhưng khi client kết nối, nó gửi probe request chứa SSID - dễ dàng bắt bằng Wireshark hoặc aircrack-ng. Thực tế còn kém bảo mật hơn vì client phải probe liên tục, tiết lộ SSID ở mọi nơi.

Đúng

WiFi security thực sự: WPA3 (hoặc WPA2 với password mạnh), 802.1X cho enterprise, VLAN riêng cho guest WiFi, disable WPS, firmware AP cập nhật. Hidden SSID chỉ gây bất tiện cho user mà không tăng security.

🗺️ Nmap Commands

Quét cơ bản

Phát hiện host và port mở nhanh

# Ping sweep - tìm host online nmap -sn 192.168.1.0/24 # Quick scan top 1000 ports nmap 192.168.1.1 # Scan tất cả 65535 ports nmap -p- 192.168.1.1 # Scan port cụ thể nmap -p 22,80,443,8080 192.168.1.1

Service & OS Detection

Xác định software và hệ điều hành

# Service version detection nmap -sV 192.168.1.1 # OS detection nmap -O 192.168.1.1 # Aggressive scan (OS + version + script + traceroute) nmap -A 192.168.1.1 # Script vuln scan nmap --script vuln 192.168.1.1

Stealth & Advanced

Kỹ thuật quét nâng cao

# SYN scan (half-open, stealthy) nmap -sS 192.168.1.1 # UDP scan (chậm nhưng quan trọng) nmap -sU --top-ports 100 192.168.1.1 # Scan qua Tor/proxy nmap --proxies socks4://127.0.0.1:9050 # Output XML cho import nmap -oX scan.xml -A 192.168.1.0/24

NSE Scripts hữu ích

Nmap Scripting Engine cho tác vụ cụ thể

# Tìm SMB vulnerabilities nmap --script smb-vuln* -p 445 target # HTTP enum directories nmap --script http-enum -p 80 target # SSL/TLS audit nmap --script ssl-enum-ciphers -p 443 target # Brute force SSH (lab only!) nmap --script ssh-brute -p 22 target

🧱 iptables Security Rules

Base Policy - Default Deny

Chặn tất cả, chỉ mở cái cần

# Flush existing rules iptables -F iptables -X # Default deny incoming, allow outgoing iptables -P INPUT DROP iptables -P FORWARD DROP iptables -P OUTPUT ACCEPT # Allow loopback iptables -A INPUT -i lo -j ACCEPT # Allow established connections iptables -A INPUT -m state \ --state ESTABLISHED,RELATED -j ACCEPT

Allow Services

Mở port cho dịch vụ cần thiết

# Allow SSH (chỉ từ subnet cụ thể) iptables -A INPUT -p tcp --dport 22 \ -s 192.168.1.0/24 -j ACCEPT # Allow HTTP/HTTPS iptables -A INPUT -p tcp --dport 80 -j ACCEPT iptables -A INPUT -p tcp --dport 443 -j ACCEPT # Allow ICMP (ping) với rate limit iptables -A INPUT -p icmp --icmp-type echo-request \ -m limit --limit 1/s -j ACCEPT

Anti-Attack Rules

Chống scan và brute force

# Rate limit SSH connections iptables -A INPUT -p tcp --dport 22 \ -m recent --set --name SSH iptables -A INPUT -p tcp --dport 22 \ -m recent --update --seconds 60 \ --hitcount 4 --name SSH -j DROP # Block port scanning (SYN flood) iptables -A INPUT -p tcp --syn \ -m limit --limit 1/s \ --limit-burst 3 -j ACCEPT # Log dropped packets iptables -A INPUT -j LOG \ --log-prefix "IPT-DROP: "

nftables (iptables hiện đại)

Syntax mới cho netfilter

# nftables đang thay thế iptables # Tạo table và chain nft add table inet filter nft add chain inet filter input \ { type filter hook input priority 0 \; \ policy drop \; } # Allow established nft add rule inet filter input \ ct state established,related accept # Allow SSH nft add rule inet filter input \ tcp dport 22 accept

OpenSSL Testing

Kiểm tra cấu hình TLS

# Test kết nối TLS openssl s_client -connect example.com:443 # Xem certificate chain openssl s_client -connect example.com:443 \ -showcerts # Kiểm tra certificate expiry openssl s_client -connect example.com:443 \ 2>/dev/null | openssl x509 -noout -dates # Test TLS version cụ thể openssl s_client -connect example.com:443 \ -tls1_3
