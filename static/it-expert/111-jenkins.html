📚 Thân Thế & Lai Lịch

Jenkins Lão Tướng đã phục vụ CI/CD từ thời Hudson (tiền thân Jenkins, 2004). Chàng ta nhớ thời Freestyle jobs, XML configuration, và "Build Now" button. "Back in my day, CI/CD meant clicking a button and praying," chàng ta hay nói.

Khi Declarative Pipeline ra đời, chàng ta mừng rỡ: "Finally, pipeline as code!" Nhưng Groovy syntax khiến chàng ta khóc. def, closure, @NonCPS — "Đây là CI/CD hay Groovy programming course?"

Bây giờ, mỗi khi ai nói "hãy dùng GitHub Actions," chàng ta phản bác: "Jenkins chạy behind firewall, controls own data, extensible vô hạn qua plugins." Rồi chàng ta update một plugin và Jenkins crashes. "Well, almost vô hạn."

📜 Bí Kíp

1. Declarative vs Scripted Pipeline: Declarative: structured, easier to read/write, built-in steps. pipeline { agent, stages, steps, post }. Scripted: Groovy-based, full flexibility, harder to maintain. Choose Declarative cho standard workflows. Scripted chỉ khi Declarative không đủ. Shared Libraries cho reusable pipeline code.
2. Plugin Management: 2000+ plugins available. Plugin compatibility matrix check TRƯỚC update. Update plugins incrementally (one at a time, test). Plugin deprecation → find alternatives. Must-have: Blue Ocean, Docker Pipeline, Credentials, Git. Avoid: unmaintained plugins (no updates in 2+ years).
3. Distributed Builds: Controller (master): orchestration only. Agents (slaves): run builds. Docker agents cho isolated builds. Kubernetes plugin cho dynamic agents. Label-based agent assignment. Cloud agents cho auto-scaling. ĐỪNG run builds on controller — security và performance risk.
4. Security: Matrix-based security cho fine-grained permissions. Credentials plugin cho secrets management. Script approval cho Groovy sandbox. CSRF protection enabled. Agent → controller security. Audit trail plugin. Regular security updates. LDAP/OIDC integration cho authentication.
5. Pipeline Best Practices: Jenkinsfile in repository root (pipeline as code). Shared Libraries cho common functions. Parallel stages cho speed. Timeout and retry cho resilience. Input step cho manual approval. Blue Ocean cho visualization. Webhooks cho trigger.

📖 Chuyện Kể Đêm Khuya

🔌 Chuyện Plugin Phá Hủy Jenkins

Monday morning, Jenkins admin update 15 plugins cùng lúc. Restart Jenkins. Boom: ClassNotFoundException, NoSuchMethodError, plugin dependency conflicts. Jenkins dashboard shows 'Jenkins is loading...' — forever. 200 developers cannot push code. CI/CD completely stopped.

Recovery: SSH vào Jenkins server. Remove newly updated plugins từ $JENKINS_HOME/plugins/. Restart. Jenkins boots với old plugin versions. Re-install plugins ONE AT A TIME, testing between each. Total downtime: 4 giờ.

💡 Bài học: NEVER update multiple plugins simultaneously. Update one, test, verify. Check compatibility matrix. Have plugin backup (copy plugins directory trước update). Jenkins LTS cho stability. Consider migrating to cloud-native CI/CD cho reduced maintenance burden.

📜 Chuyện Groovy Sandbox Escape

Developer write Groovy pipeline code accessing System.getenv() to read server environment variables. Groovy sandbox blocks. Developer requests script approval. Admin approves without reviewing. Script reads AWS credentials from environment. Developer (unintentionally) logs credentials.

💡 Bài học: Groovy script approval is SECURITY GATE. Review EVERY script approval request. Sandbox exists cho reason — don't blindly approve. Use Credentials plugin cho secrets, not environment variables. Audit approved scripts regularly. Limit who can approve scripts.

🐳 Chuyện Jenkins Docker Agent

Team configure Docker agent cho clean build environments. Each build spins up fresh container. Works great until: 50 concurrent builds × Docker pull × Docker create = Docker daemon overwhelmed. Builds queue. Timeout. Retry. More queue. Docker OOM. All builds fail simultaneously.

Fix: local Docker registry mirror (reduce pulls). Pre-built agent images. Limit concurrent builds. Kubernetes plugin cho better scheduling. Resource limits per agent.

💡 Bài học: Docker agent = clean environment but resource-intensive. Local registry mirror cho image caching. Pre-built agent images reduce startup time. Concurrent build limits prevent resource exhaustion. Kubernetes-based agents cho better resource management.

⚠️ Những Sai Lầm Kinh Điển

1. Builds on controller: Security risk + performance bottleneck. All builds on agents. Controller for orchestration only.
2. Freestyle jobs cho complex pipelines: Freestyle jobs = UI-based, not version controlled. Pipeline as code (Jenkinsfile) for everything.
3. No backup strategy: $JENKINS_HOME contains everything. Regular backups. Thin Backup plugin or filesystem snapshots.
4. Plugin sprawl: Install every plugin that sounds useful. More plugins = more maintenance, more security surface. Minimal plugin set.
5. No update strategy: Never update Jenkins or plugins → security vulnerabilities. Regular updates with testing. LTS cho stability.
