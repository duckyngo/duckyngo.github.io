📚 Thân Thế & Lai Lịch

Giao Thức Đại Sư nhớ thời dial-up modem — tiếng kêu "bíp bíp bò bò" của modem handshake là nhạc chuông thời thơ ấu. Chàng ta bắt đầu hiểu protocols khi phải debug tại sao download file qua FTP mất 2 giờ — hóa ra modem chạy ở 14.4 Kbps thay vì 56 Kbps.

Chàng ta đã chứng kiến sự tiến hóa: HTTP/1.0 → HTTP/1.1 (keep-alive!) → HTTP/2 (multiplexing!) → HTTP/3 (QUIC, goodbye TCP!). Mỗi version giải quyết problems của version trước và tạo ra problems mới.

BGP là ngôn ngữ của internet — và cũng là nguồn gốc của nhiều "internet outage" nổi tiếng. Khi Facebook sập năm 2021, Đại Sư biết ngay: "BGP withdrawal. Again."

📜 Bí Kíp

1. TCP Deep Dive: Connection: SYN → SYN-ACK → ACK. Termination: FIN → ACK → FIN → ACK. Window sizing: controls flow, enables throughput. Congestion control algorithms: Cubic (default Linux), BBR (Google, better for WAN). TIME_WAIT: 2*MSL (usually 60s) — high connection rate = many TIME_WAIT sockets. TCP_NODELAY disable Nagle's algorithm cho latency-sensitive apps.
2. HTTP Evolution: HTTP/1.1: keep-alive, chunked transfer, nhưng head-of-line blocking. HTTP/2: binary protocol, multiplexing, header compression, server push. Nhưng TCP head-of-line blocking vẫn tồn tại. HTTP/3: QUIC over UDP, giải quyết TCP HOL blocking, built-in TLS 1.3, faster connection setup (0-RTT). Deploy HTTP/3 cho modern web apps.
3. DNS Resolution: Client → Local resolver → Root (.) → TLD (.com) → Authoritative (example.com). Caching ở mỗi level. TTL controls cache duration. CNAME vs A vs AAAA vs MX vs TXT records. DNS amplification attack: small query → large response, IP spoofing. DNSSEC: chain of trust từ root, prevent spoofing.
4. BGP — Glue of the Internet: eBGP giữa autonomous systems, iBGP trong AS. Path vector protocol — routes based on AS path. BGP hijacking: announce someone else's prefix = steal traffic. RPKI (Resource Public Key Infrastructure) cho route origin validation. BGP leak can take down portions of internet — happened multiple times.
5. TLS/SSL — Encryption Layer: TLS 1.3: faster handshake (1-RTT), removed weak ciphers. Certificate management: Let's Encrypt cho free certs, auto-renewal. HSTS headers enforce HTTPS. Certificate pinning cho mobile apps. Monitor certificate expiry — expired cert = site down.
6. QUIC — The Future: UDP-based, built-in TLS 1.3, multiplexed streams without HOL blocking. 0-RTT connection resumption. Connection migration (change IP without reconnect — mobile friendly). Used by Google, Facebook, Cloudflare. HTTP/3 = HTTP over QUIC. Adoption growing rapidly.

📖 Chuyện Kể Đêm Khuya

🌍 Chuyện BGP Leak Gây Sập Internet

Năm 2019, một ISP nhỏ ở Pennsylvania misconfigure BGP — announce routes cho hàng nghìn prefixes qua uplink. Traffic meant cho Cloudflare, Amazon, và hàng trăm networks bị re-route qua ISP nhỏ này. Result: latency spike, packet loss, nhiều services inaccessible. ISP nhỏ không có bandwidth handle traffic → packets dropped.

Fix: ISP's upstream provider nhận ra anomaly sau 2 giờ, filter routes. RPKI + ROV (Route Origin Validation) sẽ prevent nhưng adoption vẫn thấp.

💡 Bài học: BGP is trust-based — thay đổi 1 router config có thể ảnh hưởng toàn cầu. RPKI adoption quan trọng cho internet security. Monitor BGP announcements cho your prefixes. Have relationship với upstream providers cho quick response.

🔄 Chuyện HTTP/2 Head-of-Line Blocking

Team upgrade từ HTTP/1.1 lên HTTP/2 mong đợi performance boost. Kết quả: mobile users ở 3G/4G connection CHẬM HƠN với HTTP/2! Why? HTTP/2 multiplexing trên single TCP connection. Packet loss trên mobile → TCP retransmission blocks ALL streams. HTTP/1.1 với 6 parallel connections: 1 connection loss chỉ block 1/6 requests.

Solution: upgrade lên HTTP/3 (QUIC/UDP) — mỗi stream independent, packet loss chỉ ảnh hưởng affected stream.

💡 Bài học: HTTP/2 better trên reliable connections. Trên lossy connections (mobile, satellite), HTTP/2 có thể worse hơn HTTP/1.1 do TCP HOL blocking. HTTP/3/QUIC giải quyết vấn đề này. Test performance trên diverse network conditions, không chỉ fast Wi-Fi.

🔒 Chuyện Certificate Expired Ngày Chủ Nhật

SSL certificate expired Sunday 2am. Automated monitoring? Không có. Auto-renewal? Không có. Website shows 'Your connection is not private.' Users report Monday morning. Revenue loss: $50K cho e-commerce site.

Fix: install new cert, enable auto-renewal (certbot renew). Thêm: certificate expiry monitoring (check 30/14/7 days trước), automated renewal via Let's Encrypt + certbot cron, alert nếu renewal fails.

💡 Bài học: Certificate management phải automated. Let's Encrypt + certbot = free + auto-renewal. Monitor certificate expiry dates — multiple alerts trước khi expire. Never rely on manual renewal. Calendar reminders KHÔNG ĐỦ — automation is the only way.

⚠️ Những Sai Lầm Kinh Điển

1. Không hiểu TCP tuning: Default TCP settings cho general purpose. High-throughput, low-latency, high-connection workloads cần tuning. Measure first, tune specific parameters.
2. HTTP/1.1 cho modern web: HTTP/2 minimum cho modern web apps. HTTP/3 cho best mobile experience. Upgrade web server và CDN config.
3. DNS single point of failure: 1 DNS server = 1 failure point. Minimum 2 DNS servers, different networks. Use managed DNS (Route 53, Cloudflare DNS) cho reliability.
4. TLS 1.0/1.1 vẫn enabled: Deprecated, insecure. Disable TLS 1.0/1.1, require TLS 1.2 minimum, prefer TLS 1.3. Test với SSL Labs.
5. Không monitor BGP: Your prefixes could be hijacked and you won't know. BGP monitoring services (RIPE RIS, BGPStream) alert on anomalies.
6. Hardcode DNS TTL quá cao: TTL 86400 (24h) cho records that change = 24h outage during migration. Use appropriate TTLs for each record type.
