📜 Thân Thế & Lai Lịch

Trương Điện Tử bắt đầu desktop development với Visual Basic 6.0 — kéo thả button, double click để viết event handler. Simple, productive, wonderful. Rồi .NET ra đời, WPF/XAML thay Windows Forms, rồi UWP thất bại, rồi WinUI 3. Khi Electron xuất hiện (VS Code, Slack, Discord), ông vừa mừng (cross-platform desktop!) vừa buồn (Chrome + Node.js + app = 500MB installer, 2GB RAM cho todo app). Tauri ra đời như giải pháp: web frontend + Rust backend, 10x nhỏ hơn Electron. Ông đã ship apps bằng Electron, Tauri, WPF, và Qt. Mỗi stack có vị trí riêng. Motto: 'Electron cho speed-to-market, Tauri cho performance-conscious, native cho platform-specific perfection.'

🗝️ Bí Kíp

Bí Kíp #1: Electron vs Tauri — Battle of Web-based Desktop

Electron: Chromium + Node.js bundled. Ưu: Chrome DevTools, npm ecosystem, mature (VS Code, Slack, Figma). Nhược: 150MB+ app size, 200MB+ RAM idle, Chromium updates = security patches needed. Tauri: system WebView + Rust backend. Ưu: 2-10MB app size, 20-50MB RAM, Rust performance cho backend, smaller attack surface. Nhược: WebView inconsistency (WebKit trên macOS, WebView2 trên Windows), Rust learning curve, smaller ecosystem. Choose: Electron cho rapid development + large team. Tauri cho performance + small binary.

Bí Kíp #2: WPF / WinUI — Windows Native

WPF: mature, XAML-based, rich controls, data binding powerful. Still relevant cho enterprise Windows apps. WinUI 3: modern Windows UI framework, Fluent Design, WinAppSDK. XAML Islands: host WinUI controls trong WPF app (gradual migration). .NET 8 improves performance significantly. MSIX packaging cho modern deployment. Windows-only nhưng: 80% enterprise users dùng Windows.

Bí Kíp #3: Qt — True Cross-platform Native

C++ framework, native look-and-feel trên Windows/macOS/Linux. QML cho modern declarative UI. Signal/Slot mechanism cho event handling. Massive control library. Qt for Python (PySide6) cho Python developers. Commercial license $$ nhưng LGPL option available. Dùng cho: desktop apps cần native performance + cross-platform + custom widgets (video editors, CAD, embedded UI).

Bí Kíp #4: Auto-update — Giữ App Luôn Mới

Electron: electron-updater, auto-download + install. Tauri: built-in updater, signed updates. Windows: MSIX auto-update, ClickOnce, Squirrel.Windows. macOS: Sparkle framework. Strategy: check update on app start, download background, prompt restart. Silent update (apply on next launch) vs prompted update. Delta updates: chỉ download changed files — faster, less bandwidth.

Bí Kíp #5: System Integration

System tray: persistent background app (notifications, quick actions). File associations: double-click .myapp file → open in your app. Deep links: protocol handler (myapp://open). Global shortcuts: keyboard shortcuts hoạt động kể cả khi app không focus. Native notifications: OS-level notifications (Action Center Windows, Notification Center macOS). Auto-start: launch on system startup (user opt-in only).

🌙 Chuyện Kể Đêm Khuya

📖 Chuyện 1: Electron RAM Eating — Khi Todo App Cần 2GB

Team build project management tool bằng Electron. MVP: 200MB installer, 800MB RAM khi mở. Manager: 'Tại sao todo app cần RAM nhiều hơn Photoshop?' Answer: Chromium runtime. Optimize: lazy load windows, offscreen rendering for non-visible content, shared process cho multiple windows. RAM giảm từ 800MB → 400MB. Vẫn nhiều hơn native app equivalent (50MB). Team accept trade-off: development speed + cross-platform > memory efficiency. Rival team build similar app bằng Tauri: 8MB installer, 60MB RAM. Feature parity. Performance comparable. Development time: 30% longer (Rust learning curve). Long-term: lower infrastructure cost, happier users.

💡 Bài học: Electron memory overhead = Chromium runtime (~150MB base). Optimization: 1) Minimize window count, 2) Offload heavy computation to Node.js process (not renderer), 3) Lazy load features, 4) Use Chrome DevTools Memory profiler. If memory is critical: Tauri, native, hoặc Qt.

📖 Chuyện 2: Tauri Bundle Size Win — 8MB vs 200MB

App cần desktop distribution: installer download từ website. Electron build: 200MB installer. Tauri build: 8MB installer. CDN cost: Electron = 200MB × 100,000 downloads = 20TB bandwidth. Tauri = 8MB × 100,000 = 800GB. Cost difference: significant ở scale. User experience: 200MB download trên slow connection = 5 phút wait. 8MB = 10 seconds. Installation abandonment rate: Electron 15%, Tauri 2%. Trade-off: Tauri development cần Rust cho backend logic. Frontend (React/Vue/Svelte) giống nhau. Team invest 2 tháng học Rust, ROI positive sau 6 tháng.

💡 Bài học: Bundle size matters cho distribution: download time, CDN cost, installation abandonment, disk usage. Tauri: 10-20x smaller than Electron. Native: similar size to Tauri. Consider distribution channel: web download (size matters), enterprise deployment (IT manages), app store (size limits).

📖 Chuyện 3: Cross-platform WebView Inconsistency

Tauri app: macOS dùng WebKit (Safari engine), Windows dùng WebView2 (Edge/Chromium). CSS backdrop-filter: hoạt động trên macOS, không trên Windows WebView2 old version. CSS container queries: supported trên WebView2, KHÔNG trên older WebKit. Result: app trông khác nhau trên mỗi OS. Fix: 1) Feature detection thay vì UA sniffing, 2) CSS fallbacks cho unsupported features, 3) Minimum WebView version requirements, 4) Test trên TẤT CẢ target OS. Lesson: Tauri's 'use system WebView' = lighter weight nhưng inconsistency risk.

💡 Bài học: Tauri system WebView = lighter but inconsistent. Electron bundled Chromium = heavier but consistent. If UI consistency critical: Electron (or enforce minimum WebView versions). If bundle size critical: Tauri + thorough cross-OS testing + CSS fallbacks.

💀 Sai Lầm Kinh Điển

❌ Sai Lầm #1: Ship debug build

Debug build: no code optimization, verbose logging, larger size, slower. Electron: development mode bundles source maps + DevTools. LUÔN build production + test production build trước release.

❌ Sai Lầm #2: Không sign application

Unsigned app: 'Unknown developer' warning (macOS Gatekeeper), SmartScreen blocking (Windows). Users scared → don't install. Code signing certificate: Apple Developer ($99/year), Windows Authenticode (varies). Sign MỌII release.

❌ Sai Lầm #3: Memory leak trong long-running app

Desktop apps run ngày/tuần — memory leaks accumulate. Monitor memory over time, profile DOM nodes count, detached elements. Electron: BrowserWindow closed but renderer process leaked references. Test: run app 24 hours, monitor memory trend.

❌ Sai Lầm #4: Ignore system integration

Desktop app that doesn't integrate with OS = glorified web page. Add: system tray, file associations, native notifications, drag-and-drop, keyboard shortcuts. These make app feel NATIVE.

❌ Sai Lầm #5: Auto-update without rollback

Bad update deployed → users stuck → manual reinstall needed. Implement rollback: keep previous version, detect crash-on-start → auto-rollback, manual rollback option in settings.
