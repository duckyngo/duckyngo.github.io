📚 Thân Thế & Lai Lịch

Hiện anh deploy YOLO cho nhà máy sản xuất (phát hiện sản phẩm lỗi) và hệ thống giao thông (đếm xe). Khi model detect sai, anh nói: "Không phải model sai, đó là creative interpretation."

📜 Bí Kíp Nhận Diện

1

YOLO Family — Từ v1 đến v11

YOLO = single-stage detector: predict bounding boxes + classes in one pass. YOLOv8/v11 (Ultralytics): state-of-art, easy to use, nhiều sizes (nano → xlarge). RT-DETR: real-time transformer-based, tốt khi cần accuracy. Chọn model size theo deployment: yolov8n (edge/mobile, 3.2M params), yolov8m (balanced), yolov8x (max accuracy, GPU). Training: yolo train data=custom.yaml model=yolov8m.pt epochs=100 imgsz=640.

2

DETR — Transformer cho Object Detection

DETR: end-to-end, không cần anchor boxes, không cần NMS. Set prediction: predict N objects (thường 100), match với ground truth bằng Hungarian algorithm. Pros: elegant, no hand-crafted components. Cons: chậm converge (500 epochs vs YOLO 100), chậm inference. Co-DETR, DINO-DETR: improved versions, converge nhanh hơn. Dùng DETR khi accuracy quan trọng hơn speed.

3

Metrics — mAP và bạn bè

mAP@0.5: IoU threshold 0.5, lenient. mAP@0.5:0.95: average over IoU 0.5 đến 0.95, strict hơn, COCO standard. Precision: trong các detection, bao nhiêu đúng? Recall: trong các objects thật, detect được bao nhiêu? Trade-off: confidence threshold cao → precision cao, recall thấp. Chọn threshold dựa trên use case: safety-critical → recall cao, spam filter → precision cao.

4

Data Annotation & Augmentation

Annotation tools: CVAT (free, self-hosted), Label Studio (versatile), Roboflow (managed + augmentation). Quality: 1 annotator check + 1 reviewer. Minimum: 500-1000 instances per class. Augmentation: Mosaic (4 images combined), random scale, HSV jitter, horizontal flip. Cẩn thận: không flip nếu left/right matters, không rotate 180° nếu gravity matters.

5

Tiny Object Detection — Bài toán khó

Objects < 32×32 pixels khó detect. Solutions: (1) Higher resolution input (1280 thay vì 640). (2) SAHI: Slicing Aided Hyper Inference — cắt ảnh thành patches, detect trên mỗi patch, merge results. (3) Multi-scale training. (4) Feature Pyramid Network (FPN) — built-in trong YOLO. (5) Tiled training data. Năm 2026: SAHI + YOLOv8 là combo standard cho tiny object detection.

6

NMS — Non-Maximum Suppression

Khi model detect cùng 1 object nhiều lần → cần NMS loại bỏ duplicate boxes. Standard NMS: giữ box confidence cao nhất, loại boxes IoU > threshold. Soft-NMS: giảm confidence thay vì loại hoàn toàn → tốt hơn cho overlapping objects. NMS threshold: 0.5-0.7. Quá thấp → miss overlapping objects. Quá cao → duplicate detections.

📖 Chuyện Kể Đêm Khuya

👤 Chuyện 1: YOLO detect mọi thứ là "person"

Deploy YOLO cho hệ thống đếm khách trong cửa hàng. Ngày đầu: "200 khách/giờ!" Sếp vui mừng. Thực tế: 50 khách + 150 mannequins + poster quảng cáo có người + phản chiếu trong gương.

Model detect mannequin là "person" confidence 0.87. Detect poster là "person" confidence 0.72. Detect reflection là "person" confidence 0.65. Anh Mắt Thần phải thêm logic: filter theo kích thước (mannequin to hơn), vị trí (poster ở trên cao), và movement (tracking để loại objects tĩnh).

💡 Bài học: Object detection trong lab khác trong real-world. Cần: (1) Domain-specific training data (bao gồm edge cases), (2) Post-processing logic, (3) Tracking để filter false positives, (4) Test trong actual environment, không chỉ trên benchmark. Model chỉ tốt bằng data và context bạn cho nó.
🔬 Chuyện 2: Tiny object detection fail

Khách hàng cần detect ốc vít bị lỗi trên dây chuyền. Ốc vít: 8×8 pixels trong ảnh 1920×1080. YOLO mAP: 12%. Anh tăng input size lên 1280: mAP 28%. Vẫn tệ.

Solution: dùng camera zoom gần hơn (tăng object size) + SAHI (sliced inference). mAP: 78%. Cuối cùng thêm custom augmentation giả lập lighting conditions nhà máy: mAP 89%. "Đôi khi giải pháp tốt nhất không phải AI tốt hơn, mà là camera tốt hơn."

💡 Bài học: Tiny object detection cần approach khác: (1) Better hardware (camera resolution, zoom), (2) SAHI for sliced inference, (3) Higher input resolution, (4) Domain-specific augmentation. Đừng blame model khi input data không đủ tốt — garbage resolution in, garbage detection out.
📊 Chuyện 3: mAP cao nhưng production tệ

Model detect sản phẩm trên kệ siêu thị: mAP@0.5 = 92% trên test set. Deploy vào camera thực: performance cảm giác chỉ 60%. Tại sao?

Test set: ảnh chụp studio, sáng đẹp, sản phẩm xếp gọn. Production: camera góc xiên, ánh sáng fluorescent, sản phẩm bị che, reflection. Domain gap kinh điển. Fix: thu thập data từ camera thực, fine-tune, thêm augmentation giả lập conditions thực tế.

💡 Bài học: mAP trên benchmark ≠ performance production. Luôn test trên data representative cho production. Thu thập "failure cases" từ production, thêm vào training data, và iterate. ML không phải "train once, deploy forever" — đó là continuous improvement.

⚠️ Những Sai Lầm Kinh Điển

❌ Dùng mAP@0.5 rồi claim "92% accuracy": mAP@0.5 lenient. Report mAP@0.5:0.95 cho realistic assessment.
❌ Ít data per class: 50 images/class không đủ. Tối thiểu 500-1000 instances per class.
❌ Không test trên production environment: Studio ≠ real-world. Test với camera, lighting, và conditions thực tế.
❌ NMS threshold cố định: Different use cases cần different NMS thresholds. Tune theo application.
❌ Quên confidence threshold: Default 0.25 cho demo, production cần tune cho precision/recall trade-off phù hợp.
❌ Model quá lớn cho edge: YOLOv8x trên Jetson Nano = 2 FPS. Chọn model size phù hợp với hardware.
