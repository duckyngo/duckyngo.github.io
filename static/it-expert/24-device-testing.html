📜 Thân Thế & Lai Lịch

Đỗ Vạn Máy từng tin vào emulator — cho đến khi app của ông pass mọi test trên emulator nhưng crash trên Samsung Galaxy S8 do custom Samsung Camera2 implementation. Từ đó, ông bắt đầu sưu tập devices như người ta sưu tập stamps. Văn phòng ông có ngăn tủ gồm 50+ devices: iPhone SE gen 1 đến iPhone 15 Pro Max, Samsung từ J series đến S series, Xiaomi, Oppo, Vivo, OnePlus, Google Pixel, và cả một chiếc BlackBerry vì 'AI BIẾT ĐƯỢC'. Đồng nghiệp gọi ông là 'Device Hoarder' nhưng mỗi khi có bug chỉ reproduce trên specific device, họ đều chạy đến ông. Motto: 'Test trên emulator để develop nhanh. Test trên device thật để ship tự tin.'

🗝️ Bí Kíp

Bí Kíp #1: Firebase Test Lab — Google's Device Farm

Cloud-based testing trên real Android + iOS devices. Robo test: AI navigate app, find crashes automatically — zero test code needed. Instrumented tests: Espresso/XCTest run trên real devices. Game Loop test: test game scenarios. CI integration: GitHub Actions, Bitrise, CircleCI. Free tier: 10 tests/day (5 virtual, 5 physical). Dashboard: screenshots, logs, video recording, performance metrics. Tip: run Robo test trên mọi PR — nó find crashes bạn KHÔNG BIẾT.

Bí Kíp #2: BrowserStack App Live & Automate

Real devices hosted in data centers. App Live: manual testing (install app, interact). App Automate: Appium tests trên 3000+ real devices. REST API: upload APK/IPA → run tests → get results. Percy: visual testing with screenshot comparison. Network simulation: 2G, 3G, 4G, WiFi. GPS mocking. Ưu: massive device coverage, no physical device management. Nhược: latency (remote device), cost ($). Alternative: AWS Device Farm, LambdaTest, Sauce Labs.

Bí Kíp #3: Screenshot Testing — Visual Regression cho Mobile

Capture screenshots → compare với baseline → detect visual changes. Tools: Paparazzi (Android, no device needed — JVM rendering), Shot (Android, device-based), SnapshotTesting (iOS, Swift). CI integration: mỗi PR generate screenshots, compare, reviewer approve visual changes. Useful khi: UI refactoring, design system updates, localization (text overflow detection). Limitation: pixel-perfect comparison fragile — dùng fuzzy matching hoặc component-level snapshots.

Bí Kíp #4: CI/CD Integration — Automated Device Testing

Pipeline: build app → run unit tests → upload to device farm → run E2E tests trên 5-10 representative devices → report results → approve/reject PR. Device matrix: flagship (Pixel 8, iPhone 15), mid-range (Samsung A54, iPhone SE), tablet (iPad Air), old device (iPhone 11, Pixel 5). Test suite: smoke tests (login, core flow) trên mọi device, full regression trên 2-3 devices. Balance: coverage vs CI time vs cost.

Bí Kíp #5: Real Device Lab vs Cloud — Khi Nào Cần Gì

In-house device lab: zero latency, full control, specific device models, one-time cost. Tốt cho: sensor testing (NFC, BLE, camera), performance profiling, UX testing. Cloud device farm: massive variety, no maintenance, pay-per-use. Tốt cho: automated regression, compatibility testing, CI/CD integration. Hybrid recommended: 5-10 physical devices in office + cloud farm cho coverage.

🌙 Chuyện Kể Đêm Khuya

📖 Chuyện 1: Test Pass Emulator, Fail Device — Camera Bug

App dùng Camera2 API chụp ảnh. Emulator: pass mọi test. Deploy → Samsung Galaxy S21: preview stretch, capture rotated 90°, flash không trigger. Nguyên nhân: Samsung custom Camera2 implementation khác AOSP. Orientation: Samsung return EXIF orientation khác Google Pixel. Debug 1 tuần: Samsung-specific code path cho orientation, flash, preview scaling. Test lại trên 5 Samsung models → 2 models vẫn buggy (different camera hardware). Cuối cùng dùng CameraX (jetpack library) — abstract away OEM differences. After incident: mọi camera-related PR phải test trên ít nhất 3 device OEMs (Samsung, Pixel, Xiaomi) trước merge.

💡 Bài học: Camera, Bluetooth, GPS, NFC — hardware APIs KHÁC NHAU giữa OEMs. Emulator KHÔNG test hardware accurately. Solutions: 1) Dùng abstraction libraries (CameraX thay Camera2), 2) Test trên 3+ OEM devices, 3) Firebase Test Lab physical devices cho CI. Hardware features = device testing mandatory.

📖 Chuyện 2: Performance Test Misleading — Emulator vs Reality

Team benchmark app performance trên emulator: 60fps consistent, memory usage 120MB, startup time 800ms. Ship with confidence. Users report: 'App lag trên Samsung A15' (entry-level phone). Test trên Samsung A15: 20-30fps, memory usage 280MB (different memory management), startup time 3.2 seconds. App basically unusable trên low-end devices chiếm 40% target market. Root cause: emulator runs trên i7 CPU + 16GB RAM + SSD. Entry-level phone: Helio P35 + 3GB RAM + eMMC storage. 10x performance gap. Fix: performance budget per device tier, lazy loading, image downscaling cho low-RAM devices, reduced animation cho slow GPUs.

💡 Bài học: Performance trên emulator = LIES. Benchmark trên target device tier. Create performance budgets: Tier 1 (flagship): 60fps, 200MB RAM. Tier 2 (mid): 45fps, 150MB RAM. Tier 3 (low-end): 30fps, 100MB RAM. Profile trên cheapest supported device — that's your real performance.

📖 Chuyện 3: Flaky Device Farm Tests — CI Trust Crisis

Firebase Test Lab integration: 20 E2E tests chạy trên 8 devices. Pass rate: 60-70%. Mỗi run, 2-3 random tests fail trên random devices. Team re-run CI 3-4 lần cho mỗi PR. Trust eroded: 'chắc device farm flaky, merge thôi.' Root causes: 1) Network latency to test server inconsistent, 2) Device state not clean between tests (leftover data), 3) Animation timing differences between devices, 4) Implicit waits hardcoded (sleep(3000)). Fix: proper waits (Espresso idling resources, XCTest expectations), test data isolation (fresh account per test), retry with analysis (different device = potentially real bug), network mocking for API-dependent tests.

💡 Bài học: Device farm flaky tests root causes: network latency, device state pollution, timing differences, animation speed. Fix: 1) Disable animations (Developer Options), 2) Isolate test data, 3) Replace sleep() with proper waits, 4) Mock network cho deterministic results. Flaky rate > 5% = investigate, don't retry-mask.

💀 Sai Lầm Kinh Điển

❌ Sai Lầm #1: Chỉ test trên emulator

Emulator = develop, không phải validate. Camera, sensors, performance, OEM-specific behavior — tất cả khác trên device thật. Minimum: 3 real devices trong development team.

❌ Sai Lầm #2: Test trên device mới nhất only

iPhone 15 Pro = fast, large screen. Nhưng users còn dùng iPhone SE (small screen), iPhone 11 (older chip). Test trên oldest supported device — đó là reality của user.

❌ Sai Lầm #3: Không monitor real-user performance

Device farm test controlled environment. Real users: bad network, background apps eating RAM, storage full. Dùng Firebase Performance Monitoring, Sentry, DataDog cho real-user metrics.

❌ Sai Lầm #4: Manual testing only

50 devices × 20 screens × 2 orientations = 2000 combinations. Manual = impossible. Automated regression cho core flows, manual cho exploratory testing on new features.

❌ Sai Lầm #5: Ignore accessibility testing trên device

TalkBack (Android), VoiceOver (iOS) behave khác trên real device vs emulator. Test accessibility trên REAL device with actual screen reader running.
