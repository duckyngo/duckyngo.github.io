📚 Thân Thế & Lai Lịch

Đa Vân Lãng Khách là kẻ lang thang giữa các đám mây. Trong khi thiên hạ trung thành với AWS hay Azure, chàng ta đã dùng qua 15 cloud providers khác nhau — từ Hetzner ở Đức với giá rẻ đến không tưởng, đến Alibaba Cloud ở Trung Quốc với hệ sinh thái riêng.

Chàng ta bắt đầu với DigitalOcean vì "5 đô la một tháng cho VPS, rẻ hơn một ly cà phê ở Starbucks". Rồi khám phá Hetzner: "Server bare metal ở Đức, 40 euro/tháng, cấu hình như EC2 instance giá $400." Chàng ta giác ngộ: Big 3 clouds tốt cho enterprise, nhưng budget-conscious builders có nhiều lựa chọn tuyệt vời.

Sau đó chàng ta gặp Cloudflare Workers — serverless ở edge, 0ms cold start, free tier 100K requests/ngày. "Đây mới là tương lai," chàng ta tuyên bố, rồi bắt đầu evangelizing edge computing cho bất kỳ ai chịu nghe.

📜 Bí Kíp

1. Hetzner — Vua Giá Rẻ Châu Âu: Dedicated servers từ €39/tháng với specs mà AWS phải $300+. Cloud VPS từ €3.29/tháng. Vị trí: Đức, Phần Lan — tuyệt vời cho EU data residency (GDPR). Nhưng không có managed services phong phú như Big 3 — bạn phải tự quản lý nhiều hơn.
2. DigitalOcean — Developer-Friendly: App Platform cho PaaS đơn giản. Managed Kubernetes giá rẻ hơn EKS/AKS/GKE. Spaces (S3-compatible) cho object storage. Tuyệt vời cho startups và side projects. Documentation xuất sắc — tutorials của DO tốt hơn cả docs chính thức của nhiều tools.
3. Cloudflare Workers — Edge Computing Revolution: V8 isolates thay vì containers — cold start gần 0ms. KV Store cho key-value, D1 cho SQLite ở edge, R2 cho object storage (S3-compatible, egress miễn phí!). Workers AI cho inference ở edge. Kết hợp với Cloudflare CDN, DDoS protection — một ecosystem hoàn chỉnh.
4. Alibaba Cloud — Vua Cloud Châu Á: Market share #1 ở Trung Quốc. Nếu target thị trường TQ, Alibaba Cloud là lựa chọn tự nhiên — ICP license support, compliance, low latency. ECS tương đương EC2, OSS tương đương S3. Nhưng documentation tiếng Anh đôi khi... thú vị.
5. Oracle Cloud — Free Tier Hào Phóng Nhất: Always-free tier với 4 ARM cores, 24GB RAM — không phải 12 tháng, mà VĨNH VIỄN. Autonomous Database free tier. OCI Ampere instances cho ARM workloads. Oracle nỗ lực vượt bậc để attract users khỏi AWS.
6. Chọn Cloud Theo Use Case: Budget-conscious? Hetzner/DigitalOcean. Edge computing? Cloudflare. China market? Alibaba. Free tier? Oracle. Enterprise với existing Microsoft stack? Azure. Data/AI? GCP. Everything else? AWS. Không có "best cloud" — chỉ có "best fit".

📖 Chuyện Kể Đêm Khuya

🖥️ Chuyện Hetzner Tiết Kiệm $200K/Năm

Một startup SaaS đang trả AWS $25K/tháng cho workload khá đơn giản: 20 VMs, PostgreSQL, Redis, object storage. Đại Sư được mời vào để "optimize costs". Sau khi phân tích, Đại Sư đề xuất: chuyển sang Hetzner dedicated servers.

CTO hoảng: "Hetzner? Nghe như hosting thập niên 2000!" Đại Sư kiên nhẫn giải thích: Hetzner có uptime 99.9%, German engineering, dedicated servers mạnh gấp 3 lần EC2 instances cùng giá. Migration mất 2 tháng, kết quả: chi phí giảm từ $25K xuống $4K/tháng. Performance tăng 40% nhờ dedicated hardware.

💡 Bài học: Cloud Big 3 tốt cho flexibility và managed services. Nhưng nếu workload stable và team có ops capability, dedicated servers ở Hetzner/OVH có thể tiết kiệm 70-80% chi phí. Đừng để "cloud-first" trở thành "cloud-only".

⚡ Chuyện Cloudflare Workers Thay Thế Backend

Một team đang chạy API gateway trên EC2 instances — 4 servers, ALB, auto-scaling. Latency: 150ms trung bình vì server ở us-east-1 mà users ở toàn cầu. Chi phí: $800/tháng.

Đại Sư suggest: chuyển API logic sang Cloudflare Workers. Code chạy ở 300+ edge locations. Cold start: 0ms. Latency giảm xuống 20ms cho mọi users. Chi phí: $25/tháng. Team lead không tin, phải proof-of-concept 2 tuần mới chịu approve migration.

💡 Bài học: Edge computing thay đổi game. Cloudflare Workers + KV + D1 + R2 tạo thành full-stack platform ở edge. Không phải mọi workload phù hợp, nhưng cho API routing, authentication, personalization — edge is king.

🏢 Chuyện Oracle Cloud Free Tier Quá Tốt

Một homelab enthusiast phát hiện Oracle Cloud free tier: 4 ARM cores, 24GB RAM, 200GB storage — MIỄN PHÍ VĨNH VIỄN. Anh ta setup WireGuard VPN, Pi-hole, personal website, và Minecraft server cho con. Tất cả free.

6 tháng sau, Oracle "reclaim idle instances" — xóa sạch vì "compute bị coi là idle". Anh ta mất tất cả. Không backup. Gọi cho Đại Sư khóc lóc. Đại Sư nói: "Free tier thì free. Nhưng data không có backup thì vô giá — theo nghĩa xấu."

💡 Bài học: Free tier tuyệt vời để learn và experiment. Nhưng KHÔNG BAO GIỜ chạy production trên free tier mà không có backup. Oracle có quyền reclaim idle free instances. Backup data ở nơi khác — ít nhất một bản local.

⚠️ Những Sai Lầm Kinh Điển

1. Chọn cloud chỉ vì rẻ: Rẻ nhưng không có managed services, không có support, không có compliance certifications = tốn tiền ops gấp đôi. Tính Total Cost of Ownership, không chỉ sticker price.
2. Bỏ qua data egress costs: Hetzner và Cloudflare R2 có free egress. AWS, Azure, GCP charge egress nặng. Nếu workload data-heavy, egress cost có thể chiếm 40% bill.
3. Vendor lock-in với niche cloud: Build trên proprietary APIs của cloud nhỏ. Khi cloud đó shutdown hoặc pivot, bạn stuck. Prefer open standards: S3-compatible storage, Kubernetes, PostgreSQL.
4. Chạy production trên free tier: Free tier có resource limits, có thể bị reclaim, không có SLA. Dev/test thì OK. Production? Trả tiền đi.
5. Không xét data residency: EU data ở Hetzner Đức? OK. Alibaba Cloud ở Singapore cho EU customers? Cần xem xét GDPR. Data residency requirements quan trọng hơn giá.
