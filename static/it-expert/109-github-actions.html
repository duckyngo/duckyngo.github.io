📚 Thân Thế & Lai Lịch

Đại Sư nhớ thời Jenkins — Groovy pipelines, plugin hell, "works on my Jenkins." Khi GitHub Actions ra đời, chàng ta thở phào: "CI/CD native trong GitHub? YAML workflow? Marketplace actions? Yes please!"

Chàng ta migrate 50 Jenkins pipelines sang GitHub Actions trong 2 tháng. Most pipelines simpler, faster, và không cần maintain Jenkins server nữa. "I deleted our Jenkins server and nobody complained," chàng ta tự hào.

Nhưng GitHub Actions cũng có pitfalls: workflow chạy 2 giờ costs $$$, secrets leaked trong logs, self-hosted runners security risks. "Every CI/CD system has its own special brand of pain," chàng ta philosophical.

📜 Bí Kíp

1. Workflow Syntax: on: push, pull_request, schedule, workflow_dispatch. jobs → steps → uses/run. Matrix strategy cho multi-version testing. Concurrency groups prevent duplicate runs. Permissions cho GITHUB_TOKEN scoping. Environment protection rules cho production.
2. Reusable Workflows: workflow_call cho shared workflows. Inputs và secrets parameters. Central repository cho organization workflows. Version pinning (uses: org/workflow@v1). DRY principle cho CI/CD. Composite actions cho reusable step groups.
3. Self-Hosted Runners: Own infrastructure cho builds. Cost savings cho heavy CI/CD. Custom software/hardware (GPU builds). Security: runner compromise = code access risk. Ephemeral runners cho security. Runner groups cho organization management. Auto-scaling runners (actions-runner-controller cho K8s).
4. Secrets & Security: Repository secrets, environment secrets, organization secrets. GITHUB_TOKEN scoped permissions. Third-party actions audit (pin to commit SHA, not tag). Dependabot cho action updates. OIDC cho cloud provider authentication (no long-lived credentials). Secret scanning prevent commit.
5. Performance Optimization: Cache action cho dependencies (node_modules, pip cache). Artifact upload/download cho cross-job data. Concurrency groups cancel redundant runs. Matrix strategy cho parallel testing. Self-hosted runners cho custom hardware. Timeout settings prevent runaway jobs.

📖 Chuyện Kể Đêm Khuya

💸 Chuyện Workflow 2 Giờ Cost $$$

CI pipeline: build → test → deploy. Build: 15 minutes. Test suite: 1 giờ 45 phút (integration tests hit real database). Cost: $0.008/minute × 120 minutes = $0.96/run. 50 PRs/day × 3 runs each = 150 runs/day. Monthly: $4,320. CTO: 'CI costs more than our cloud infrastructure!'

Optimization: parallelize test suites (matrix strategy). Cache dependencies. Skip unchanged modules. Move integration tests to nightly run. PR tests: unit tests only (10 minutes). Result: $0.08/run, monthly: $360. 92% reduction.

💡 Bài học: CI costs scale with team size and frequency. Monitor GitHub Actions usage. Parallelize tests. Cache aggressively. Right-size test scope per trigger (PR = fast, main = full). Self-hosted runners cho heavy workloads. Timeout cho runaway jobs.

🔐 Chuyện Secret Bị Lộ Trong Logs

Workflow step: echo $DATABASE_URL cho debugging. GitHub masks secrets in logs — nhưng chỉ exact match. DATABASE_URL chứa password URL-encoded. URL-decoded version NOT masked. Attacker reading workflow logs extract password.

Worse: third-party action logs environment variables verbosely. Secrets in env vars exposed. Fix: audit all workflow logs. Remove debug outputs. Use add-mask cho custom secrets. Audit third-party actions cho logging behavior.

💡 Bài học: GitHub masks exact secret values in logs, NOT transformations. Don't echo secrets. Don't pass secrets as command arguments (visible in process listing). Use step environment variables instead of workflow-level. Audit third-party actions. Minimize secret exposure.

🏃 Chuyện Self-Hosted Runner Compromise

Self-hosted runner trên shared VM. Developer A's workflow installs malicious dependency. Dependency persists on runner. Developer B's workflow runs on same runner — malicious code executes in context of different repository. Cross-repository contamination.

Fix: ephemeral runners (destroy after each job). Containerized runners. Dedicated runners per repository/team. Runner groups with access control. Regular security audits.

💡 Bài học: Shared self-hosted runners = shared security context. Ephemeral runners (recreate each job) cho security. Container-based runners cho isolation. actions-runner-controller cho Kubernetes-based auto-scaling ephemeral runners. NEVER share runners between different trust levels.

⚠️ Những Sai Lầm Kinh Điển

1. Pin actions to tag (not SHA): uses: actions/checkout@v3 — tag có thể bị overwrite. Pin to SHA: uses: actions/checkout@abc123. Dependabot update SHAs automatically.
2. GITHUB_TOKEN với quá nhiều permissions: Default permissions broad. Set permissions: read-all + specific writes needed. Principle of least privilege cho tokens.
3. No concurrency control: PR push 5 commits = 5 workflow runs. Concurrency group cancel redundant runs. Save time and money.
4. Large artifacts: Upload 2GB artifact each run. Storage costs accumulate. Retention days: 1-90 (default 90). Set appropriate retention.
5. No timeout: Workflow hung → runs until 6-hour default timeout. Set timeout-minutes per job. 30 minutes reasonable for most builds.
