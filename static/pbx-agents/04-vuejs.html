Bi Kip Vue.js

Nhung dieu ta hoc duoc sau bao nhieu lan "deploy xong moi phat hien bug"
1
ref() vs reactive() — Chon sai la kho ca doi
ref() cho primitive values (string, number, boolean) va khi can .value de truy cap. reactive() cho object/array nhung KHONG BAO GIO destructure truc tiep, vi se mat reactivity. Meo: cu dung ref() cho moi thu, an toan hon nhieu. reactive() chi dung khi ban thuc su hieu minh dang lam gi.
// SAI - mat reactivity khi destructure
const state = reactive({ count: 0 })
const { count } = state // count khong reactive nua!

// DUNG - dung ref() hoac toRefs()
const count = ref(0) // luon reactive
const { count } = toRefs(state) // van reactive
2
Composables — Vu khi bi mat cua Composition API
Dung tao component khong lo voi 500 dong code trong setup(). Hay tach logic ra thanh cac composable function (useXxx). Day la ly do Composition API ton tai — de ban co the tai su dung logic giua cac component ma khong can mixins (cai thu doc hai do).
// composables/useCallStatus.ts
export function useCallStatus() {
  const isOnCall = ref(false)
  const duration = ref(0)
  const startCall = () => { isOnCall.value = true }
  return { isOnCall, duration, startCall }
}
3
watchEffect() vs watch() — Hai anh em khac tinh
watchEffect() tu dong theo doi moi dependency ben trong va chay ngay lap tuc. watch() chi theo doi nhung gi ban chi dinh va cho ban gia tri cu + moi. Meo: watchEffect() cho "toi muon lam gi do khi bat ky dependency nao thay doi". watch() cho "toi muon biet cu the cai gi thay doi va gia tri truoc la bao nhieu".
4
Template Refs + nextTick() — Doi DOM san sang
Khi ban thay doi reactive data, DOM khong cap nhat ngay lap tuc. Vue "gom" cac thay doi lai va cap nhat mot lan. Neu ban can truy cap DOM ngay sau khi thay doi data, hay dung nextTick(). Day la sai lam kinh dien cua nguoi moi: thay doi data roi doc DOM ngay — va tu hoi tai sao no van la gia tri cu.
const message = ref('Xin chao')
message.value = 'Tam biet'

// SAI: DOM chua cap nhat
console.log(el.textContent) // van la "Xin chao"

// DUNG: doi nextTick
await nextTick()
console.log(el.textContent) // "Tam biet"
5
provide/inject — Chong Prop Drilling
Khi ban truyen props qua 5-6 tang component, code se tro nen nhu cai cay Giang sinh — dep nhung roi. provide/inject cho phep component ong truyen thang data cho component chau ma khong can qua trung gian. Nhung nho: dung lam dung, vi debug se kho hon khi ban khong biet data den tu dau.
6
defineModel() — Vuot cua Vue 3.4+
Truoc Vue 3.4, two-way binding cho component tu lam la cuc hinh: defineProps + defineEmits + computed get/set. Gio chi can defineModel() la xong. Mot dong thay ba muoi dong. Day la minh chung rang Vue luon tien hoa de dev bot kho.
// Truoc Vue 3.4 - dai dong dong
const props = defineProps(['modelValue'])
const emit = defineEmits(['update:modelValue'])
const value = computed({
  get: () => props.modelValue,
  set: (v) => emit('update:modelValue', v)
})

// Vue 3.4+ - gon nhu tho
const value = defineModel<string>()

Cai Bay Reactivity — Hay Cach Ong Mat 3 Ngay Debug

Ngay xua, ong co mot cai object lon nhu cai nha, ong bo no vao reactive() dep de lam. Roi mot ngay, ong can truyen mot phan cua no cho component con. Ong vui ve destructure: const { name, age } = userState.

Ong thay doi name, nhung UI cu nhu tuong — khong nhuc nhich. Ong tuong Vue bi loi. Ong doc lai docs. Ong hoi Stack Overflow. Ong thuc den 3 gio sang.

Hoa ra, khi destructure reactive object, cac gia tri primitive bi "tach roi" khoi Proxy va mat het reactivity. Ong phai dung toRefs() hoac storeToRefs() voi Pinia store. Tu do, ong khong bao gio destructure reactive object nua. Ong bao con chau: "Dung bao gio xat nho reactivity ra!"

💡 Bai hoc: Destructure reactive object = mat reactivity. Dung toRefs() hoac cu truy cap qua object goc.
⚔

Cuoc Chien v-if vs v-show — Ai Moi La Vo Lam Minh Chu?

O mot du an ngay xua, team ong co hai phe: phe v-if va phe v-show. Phe v-if noi: "Toi tiet kiem DOM node, toi la vuong!" Phe v-show dap: "Toi toggle nhanh hon, toi moi la vuong!"

Ho tranh luan 2 tieng trong cuoc hop. PM ngoi do khong hieu gi, chi biet la deadline sap den.

Cuoi cung ong phai dung day phan xu: v-if dung khi condition it thay doi (vi no destroy/recreate DOM). v-show dung khi toggle thuong xuyen (vi no chi thay doi CSS display). Trong PBX project, cai call panel toggle lien tuc thi dung v-show. Cai admin settings hiem khi mo thi dung v-if.

Tu do hai phe hoa binh. Nhung ong van thay ho lien nhau ve v-if voi v-for tren cung element... do lai la chuyen khac.

💡 Bai hoc: v-if cho render co dieu kien (lazy), v-show cho toggle thuong xuyen. Va DUNG BAO GIO dung v-if voi v-for tren cung element!
🎃

Ac Mong Prop Drilling — Khi Props Di Qua 7 Tang Dia Nguc

Ong con nho cai du an dau tien dung Vue. Ong co mot cai theme color can truyen tu App.vue xuong den mot cai button nho xiu nam sau tan 7 tang component.

Ong truyen props: App → Layout → Sidebar → Menu → MenuItem → SubMenu → Button. Moi component phai khai bao defineProps roi truyen tiep xuong. Code dai nhu song Han.

Mot ngay, PM noi: "Doi ten prop tu 'themeColor' thanh 'primaryColor' nhe." Ong phai sua 7 file. Ong muon khoc.

Roi ong hoc duoc provide/inject. Ong provide o App.vue, inject o Button — xong. Sau nay co Pinia, moi thu con de hon nua. Ong dat theme vao store, component nao can thi lay. Doi ong khong bao gio drilling nua.

💡 Bai hoc: Qua 2-3 tang props thi nen dung provide/inject hoac Pinia store. Prop drilling la duong dan ngan nhat den burnout.
👻

Ma onUnmounted — Khi Memory Leak Am Tham An Mung

Co mot lan, ong lam tinh nang realtime cho PBX — lang nghe SIP events qua WebSocket. Ong dang ky event listener trong onMounted(), roi... quen mat onUnmounted().

Moi lan user navigate di roi quay lai, mot listener moi duoc tao ra ma cai cu khong bi huy. Sau 30 phut, tab browser an 2GB RAM. User goi dien bao: "Sao may tinh tui nong nhu lo nuong vay?"

Tu do ong co quy tac bat thanh van: "Co onMounted thi phai co onUnmounted. addEventListener thi removeEventListener. Subscribe thi unsubscribe. Nhu am voi duong, co vay co lai."

💡 Bai hoc: LUON don dep trong onUnmounted(). setInterval, addEventListener, WebSocket subscription — tat ca phai duoc huy khi component bi destroy.

Nhung Loi Kinh Dien

Nhung "ho" ma ai cung phai roi xuong it nhat mot lan
💥
Quen .value voi ref()
Trong script setup, ref() can .value de doc/ghi. Trong template thi khong can. So luong lan dev viet count++ thay vi count.value++ roi tu hoi "tai sao no khong tang?" nhieu khong dem xue.
✅ Meo: bat ESLint + Vue plugin, no se nhac ban ngay.
😱
Thay doi prop truc tiep
Props la ONE-WAY data flow. Thay doi prop truc tiep se bi Vue canh bao va gay ra bug kho debug. Dung lam cha no lam, dung sua hang xom.
✅ Dung emit event len parent hoac dung defineModel() cho two-way binding.
🔥
v-for khong co :key
Khong dung :key hoac dung index lam key. Vue se khong biet element nao thay doi, dan den render sai va bug "ma" xuat hien khi sort/filter list.
✅ Luon dung unique ID lam key. KHONG BAO GIO dung index neu list co the thay doi.
👀
Goi API trong computed()
computed() la synchronous va duoc cache. Goi async API trong computed la hanh dong pham phap. No se chay nhieu lan khong kiem soat va khong bao gio return dung.
✅ Goi API trong onMounted() hoac watch(), luu ket qua vao ref().
🚨
Deep watch ma khong can thiet
watch(obj, cb, { deep: true }) tren object lon se lam Vue phai duyet toan bo object moi khi co bat ky thay doi nao. Performance chet nhu con ho bi cam.
✅ Chi watch property cu the: watch(() => obj.value.name, cb). Chinh xac va nhanh hon.
🤯
Component khong lo 1000+ dong
Nhoi het moi thu vao mot component duy nhat. Template 300 dong, script 500 dong, style 200 dong. Doi ma maintain code nay bat dau tu khi ban khong nho minh viet gi.
✅ Tach thanh composables + child components. Moi component nen duoi 200 dong.

Vue.js Trong Du An PBX

Cach Vue 3.5 tro thanh xuong song cua giao dien PBX

Toan bo giao dien nguoi dung cua he thong PBX duoc xay dung tren Vue 3.5 voi Composition API va TypeScript. Day la mot he thong quan ly tong dai dien thoai doanh nghiep, noi ma moi giay deu quan trong — tu viec hien thi trang thai cuoc goi realtime, quan ly hang doi, den giao dien agent xử ly cuoc goi.

📞
Giao Dien Cuoc Goi Realtime
Vue reactivity system la nen tang de hien thi trang thai cuoc goi (ringing, connected, on-hold) theo thoi gian thuc. ref() va computed() giup UI cap nhat tuc thi khi SIP events thay doi.
🌳
Pinia Quan Ly State
Trang thai agent (available, busy, away), danh sach cuoc goi, cau hinh he thong — tat ca duoc quan ly tap trung trong Pinia stores. storeToRefs() dam bao reactivity khi destructure store data.
🛣
Vue Router & Guards
Dieu huong giua cac trang: Dashboard, Call History, Agent Management, Settings. Navigation guards kiem tra quyen truy cap va trang thai dang nhap truoc khi cho phep vao trang.
📈
DevExtreme Components
Bang du lieu, bieu do thong ke, form nhap lieu — DevExtreme cung cap UI components san co. Vue 3 wrapper giup tich hop mượt ma voi Composition API va reactive data binding.
🔌
Composables Cho Logic Nghiep Vu
useCallManager(), useAgentStatus(), useWebSocket() — logic nghiep vu duoc dong goi trong composables, tai su dung across components. Clean code, de test, de maintain.
🔷
TypeScript An Toan
Moi component, moi composable, moi store deu co type definitions. defineProps<T>() va defineEmits<T>() dam bao rang data flow giua cac component luon dung kieu, bat loi ngay luc compile thay vi luc production.
