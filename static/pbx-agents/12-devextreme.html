Hành trình từ jQuery đến DevExtreme Đại Sư

Thưở xưa, ở một vùng đất nọ có tên là "Enterprise Vietnam", có một lão ông suốt ngày ngồi code bảng dữ liệu. Người ta gọi lão là Giao Diện Đại Sư — không phải vì lão đẹp trai (mặc dù cũng không xấu), mà vì lão có khả năng biến bất kỳ cái bảng Excel nào thành một DataGrid hoành tráng chỉ trong vài dòng config.

Lão bắt đầu cuộc đời với jQuery và nhứng cái table HTML xấu xí. Rồi một ngày đẹp trời, lão phát hiện ra DevExtreme — bộ thư viện UI enterprise mạnh nhất mà lão từng thấy. Từ đó, lão tu luyện suốt ngày đêm, nghiên cứu từng component, từ DataGrid huyền thoại đến TreeView thần bí, từ Form linh hoạt đến Popup đa dạng.

Lão thành thạo custom templates, data sources từ remote API, virtual scrolling cho hàng chục ngàn dòng, và export ra Excel/PDF mượt như nước.

Giờ đây, lão gia nhập dự án PBX — một hệ thống tổng đài doanh nghiệp cần giao diện mạnh mẽ, linh hoạt, và quan trọng nhất là phải chạy nhanh. Lão nói: "Grid ta render, form ta validate, tree ta expand — thiên hạ vô địch!"

Bí Kíp Giao Diện Đại Sư

Những chiêu thức bí truyền chỉ truyền cho người có duyên
1

DataGrid remoteOperations — Để server lo hết!

Đừng bao giờ load 10,000 dòng về client rồi filter. Hãy bật remoteOperations: true và để backend xử lý sorting, filtering, paging. Grid chỉ hiển thị — nhanh như chớp. Ông nội dạy: "Việc nặng thì giao cho server, con cháu nhàn thân!"

2

Virtual Scrolling + Row Rendering Mode

Bật scrolling.mode: 'virtual' kết hợp rowRenderingMode: 'virtual'. Grid sẽ chỉ render những row đang hiển thị trên màn hình. 100K rows mà scroll mượt như đang lướt TikTok. Bí kíp này đã cứu biết bao dự án khỏi cảnh lag!

3

CustomStore — Vũ khí tối thượng kết nối API

Đừng dùng ArrayStore rồi tự fetch data. Hãy dùng CustomStore với load, insert, update, remove functions. DevExtreme sẽ tự quản lý state, caching, và retry. Viết một lần, dùng mãi mãi — như tình yêu với code vậy!

4

Template Slot trong Vue — Sức mạnh thật sự

DevExtreme Vue cho phép dùng #cell-template slot để custom render từng cell. Kết hợp với Vue's reactivity, bạn có thể render components phức tạp bên trong grid mà không lo performance. Nhớ dùng :key binding để tránh re-render toàn bộ!

5

State Persistence — Người dùng sẽ cảm ơn bạn

Bật stateStoring để lưu trạng thái column order, filter, sort vào localStorage. Lần sau user mở lại, mọi thứ y chang lúc trước. Khách hàng sẽ nghĩ bạn là phù thủy — mà thực ra chỉ là 3 dòng config!

6

Export có style — Đừng export trần trụi

Khi export Excel, dùng onExporting + exceljs để custom header, thêm logo công ty, format cell colors. Sếp nhận file Excel mà tưởng đang xem báo cáo từ SAP chứ không phải từ web app. Ấn tượng gấp 10 lần!

7

repaintChangesOnly — Chống giật lag thần thánh

Set repaintChangesOnly: true trên DataGrid. Thay vì re-render toàn bộ grid khi data thay đổi, chỉ render lại cells đã thay đổi. Tiết kiệm CPU, tiết kiệm điện, cứu trái đất — vừa code vừa bảo vệ môi trường!

Chuyện Kể Bên Bàn Code

Những câu chuyện bi hài mà ông nội kể lại, có thật 100%... có thể
💀
Chương 1: "10,000 Dòng Và Cái Tab Browser Tắt Thở"

Ngày xưa, có một chàng dev trẻ tuổi nghĩ rằng DataGrid có thể handle mọi thứ. Chàng quyết định load 10,000 dòng call history từ API về client, không paging, không virtual scrolling, không remoteOperations — vì chàng tin vào sức mạnh của Chrome.

"Chrome có 16GB RAM mà lo gì? Cứ load hết về!" — Lời cuối của chàng dev trước khi tab crash.

Grid render xong thì Chrome hiện ra cái popup huyền thoại: "This page is not responding. Kill page?". PM đứng sau lưng, mặt tái nhợt. QA cười không nhặt được mồm.

Từ đó, chàng dev ngộ ra chân lý: bật virtual scrolling, set pageSize: 50, enable remoteOperations. Grid load trong 200ms. PM hạnh phúc, QA hết việc, Chrome thở phào.

Bài học: Đừng bao giờ đánh giá thấp sức phá hoại của 10K DOM nodes. Virtual scrolling là bạn, không phải là option.
🚨
Chương 2: "Cái Popup License Xuất Hiện Trước Mặt Khách Hàng"

Hôm đó là ngày demo sản phẩm cho khách hàng Hàn Quốc. Mọi thứ đã sẵn sàng: slide đẹp, data test đầy đủ, dev ăn mặc lịch sự (thậm chí còn cạo râu).

Bấm vào trang quản lý cuộc gọi... DataGrid hiện ra hoành tráng. Khách hàng gật đầu hài lòng. Rồi đột nhiên, một cái popup đỏ lòm xuất hiện giữa màn hình:

"DevExtreme: The license key is invalid. Please contact DevExpress to obtain a valid license."

Cả phòng im lặng. PM nhìn dev. Dev nhìn PM. Khách hàng nhìn cả hai rồi hỏi: "이게 뭐예요?" (Cái gì vậy?)

Hóa ra ai đó commit file .env mới mà quên copy license key. Từ đó, team có rule: kiểm tra license trước MỌI buổi demo, và license key được lưu ở vault, không phải ở file .env.example.

Bài học: License DevExtreme không phải thứ bạn nhớ lúc nào cũng được. Hãy kiểm tra nó như kiểm tra xem mình có quên mặc quần trước khi ra đường.
⚔️
Chương 3: "Cuộc Chiến CSS Override Kéo Dài 3 Ngày"

Designer gửi mockup: "Em muốn cái header của DataGrid có border-radius 12px, font chữ custom, và gradient background."

Dev: "OK, dễ mà!"

30 phút sau: .dx-datagrid-headers — không ăn. 2 giờ sau: .dx-datagrid .dx-header-row — override bị đè. 1 ngày sau: .dx-datagrid-headers .dx-datagrid-table .dx-row .dx-header-row td — vẫn thua.

"Tại sao CSS DevExtreme lại có specificity cao như vậy?! Ai thiết kế cái này?! !important đâu, cho ta !important!" — Dev lúc 2h sáng, mắt đỏ hoe.

Cuối cùng, sau 3 ngày chiến đấu, dev phát hiện ra bí mật: DevExtreme dùng inline styles cho một số property. Giải pháp? Dùng onCellPrepared event để set style bằng JS, hoặc dùng cssClass prop kết hợp encapsulation. Hoặc đơn giản hơn: dùng DevExtreme's ThemeBuilder để custom theme từ đầu.

Bài học: Đừng đánh nhau với DevExtreme CSS bằng tay. Hãy dùng ThemeBuilder hoặc API events. Chiến tranh CSS chưa bao giờ có người thắng.
🤯
Chương 4: "Cái TreeView Đệ Quy Vô Hạn"

Có lần, một dev thiết kế cây phòng ban cho PBX. Data từ backend trả về dạng flat array với parentId. Dev bind vào TreeView, chạy thử — cây hiện ra đẹp lắm. 3 cấp, gọn gàng.

Rồi đến khi test với data thật. Ai đó trong database set parentId của phòng ban A trỏ về phòng ban B, mà phòng ban B lại trỏ về phòng ban A. Circular reference!

"Sao cái TreeView cứ expand mãi không dừng? Ơ... ơ... Chrome lại crash rồi!" — Dev, lần thứ 2 trong đời thấy Chrome tắt thở.

Giải pháp: validate data trước khi bind, dùng dataStructure: 'tree' thay vì 'plain' khi cần, và luôn có depth limit. Ông nội dạy rằng: "Cây nào cũng có gốc, đệ quy nào cũng cần điểm dừng!"

Bài học: Luôn validate dữ liệu cây. Circular reference + recursive rendering = máy tính thành lò sưởi.

Sai Lầm Kinh Điển

Những cái bẫy DevExtreme mà 99% dev đều từng mắc (kể cả Đại Sư hồi trẻ)
1
Load toàn bộ data về client
Gọi API lấy hết 50K records rồi nhét vào ArrayStore. Grid lag, browser khóc, user bỏ đi uống cà phê chờ load.
Dùng CustomStore + remoteOperations. Server trả về từng page 50 records. Nhanh, gọn, lẹ!
2
Quên tắt autoExpandAll trên TreeList
TreeList có 500 nodes, autoExpandAll: true mặc định. Mở trang lên là thấy cây phả hệ 20 đời hiện ra cùng lúc.
Set autoExpandAll: false, dùng expandedRowKeys để control, hoặc lazy load children.
3
CSS !important chiến tranh
Override DevExtreme CSS bằng 500 dòng !important. Mỗi bản update DX mới là CSS vỡ hết, phải đi fix lại toàn bộ.
Dùng ThemeBuilder tạo custom theme. Hoặc dùng cssClass prop + onCellPrepared event.
4
Không set keyExpr trên DataGrid
Quên set keyExpr, grid không biết row nào là row nào. Selection loạn, editing sai row, master-detail mở nhầm chỗ.
LUÔN set keyExpr (VD: keyExpr="id"). Đây là dòng config quan trọng nhất mà hay bị quên nhất!
5
Bind reactive object trực tiếp vào dataSource
Dùng Vue ref/reactive trực tiếp làm dataSource. Mỗi lần state thay đổi, grid re-render toàn bộ, giao diện nhấp nháy như đèn disco.
Wrap trong computed hoặc dùng DataSource instance. Set repaintChangesOnly: true để chỉ update cells thay đổi.
6
Popup trong Popup trong Popup
Mở popup chỉnh sửa, bên trong có popup chọn nhân viên, bên trong có popup confirm. Matryoshka doll edition. Z-index: 999999.
Thiết kế lại flow. Dùng drawer/panel thay popup con. Tối đa 2 levels popup, còn lại dùng inline editing.
7
Quên dispose DataSource khi unmount
Component bị unmount nhưng DataSource vẫn đang polling/fetching. Memory leak tích tụ, app chạy càng lâu càng chậm.
Gọi dataSource.dispose() trong onBeforeUnmount(). Hoặc dùng composable pattern để tự động cleanup.

DevExtreme Trong Dự Án PBX

Vì sao PBX chọn DevExtreme làm vũ khí giao diện chính

Dự án PBX — hệ thống tổng đài doanh nghiệp — cần giao diện có khả năng hiển thị lượng lớn dữ liệu real-time: lịch sử cuộc gọi, danh sách agents, cây phòng ban, cấu hình hệ thống. DevExtreme 23.2 kết hợp Vue.js là lựa chọn hoàn hảo vì nó sinh ra để xử lý enterprise UI — nơi mà "bảng dữ liệu là vua" và "form cấu hình là hoàng hậu".

📞

Lịch Sử Cuộc Gọi (Call History)

DataGrid với virtual scrolling hiển thị hàng chục ngàn cuộc gọi. Filtering theo ngày, agent, trạng thái. Export báo cáo Excel cho quản lý. Remote operations để backend xử lý query nặng.

🏗️

Cây Phòng Ban & Tổ Chức

TreeView/TreeList hiển thị cấu trúc phòng ban, nhóm agent, hàng đợi cuộc gọi. Drag & drop để di chuyển agent giữa các nhóm. Lazy loading cho tổ chức lớn.

⚙️

Form Cấu Hình Hệ Thống

DevExtreme Form với dynamic layout cho màn hình cấu hình: thông tin agent, cài đặt hàng đợi, lịch làm việc, kịch bản cuộc gọi. Validation rules đảm bảo data luôn hợp lệ.

📊

Dashboard & Báo Cáo

DataGrid grouped views cho thống kê cuộc gọi theo agent, theo phòng ban, theo giờ. Summary rows tính tổng, trung bình, tỷ lệ cuộc gọi nhỡ. PivotGrid cho phân tích đa chiều.

🔔

Popup Thông Báo & Confirm

Popup hiển thị thông tin cuộc gọi đến, xác nhận chuyển cuộc gọi, cảnh báo hàng đợi đầy. Toast notifications cho sự kiện real-time từ WebSocket.

🎛️

Quản Lý Agent & Extension

DataGrid với inline editing để quản lý danh sách agent, số nội bộ (extension), trạng thái. Batch editing cho thao tác hàng loạt. Master-detail view cho chi tiết từng agent.
