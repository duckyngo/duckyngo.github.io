📚 Thân Thế & Lai Lịch

Đại Sư chọn GitLab vì "all-in-one platform." Git repository + CI/CD + Container Registry + Issue Tracking + Wiki + Security Scanning — tất cả built-in. Không cần integrate 10 tools khác nhau.

Chàng ta viết .gitlab-ci.yml đầu tiên: 3 stages (build, test, deploy), 5 jobs, YAML anchors cho DRY. Pipeline runs, tests pass, deploy to staging automatic. "DevOps in 30 lines of YAML," chàng ta tự hào.

6 tháng sau: .gitlab-ci.yml 500 dòng, 15 stages, 30 jobs, include templates từ 5 repositories. Pipeline takes 45 phút. Runner queue backed up. Cache miss rebuilds everything. "DevOps in 500 lines of YAML and 45 minutes of waiting," chàng ta thở dài.

📜 Bí Kíp

1. Pipeline Architecture: Stages run sequentially, jobs trong stage run parallel. DAG (needs keyword) cho custom dependencies. Parent-child pipelines cho modularity. Multi-project pipelines cho cross-repo. Rules cho conditional job execution. Workflow rules cho conditional pipeline creation.
2. Runners — Execution Environment: Shared runners (GitLab.com provided). Group runners cho organization. Project runners cho specific needs. Docker executor: isolated containers. Kubernetes executor: pods per job. Shell executor: direct on machine (security risk). Auto-scaling: Docker Machine hoặc Kubernetes.
3. Cache & Artifacts: Cache: speed up jobs (dependencies, build tools). Per-branch caching. Cache key strategies: files hash, branch name. Artifacts: pass data between jobs/stages. Artifact expiration: set retention. ĐỪNG cache build outputs — use artifacts. ĐỪNG artifact dependencies — use cache.
4. Environments & Deployments: environment keyword cho deployment tracking. Dynamic environments per branch (review apps). Auto-stop environments after merge. Protected environments cho production (manual approval). Deployment history và rollback. Environment-specific variables.
5. Templates & Includes: include:template cho GitLab templates. include:project cho cross-project templates. include:remote cho external URLs. YAML anchors và extends cho DRY. Component catalog (newer feature) cho reusable CI/CD components. Centralize common job definitions.

📖 Chuyện Kể Đêm Khuya

⏰ Chuyện Pipeline 45 Phút

GitLab CI pipeline: build (10 min) → lint (5 min) → unit tests (15 min) → integration tests (10 min) → deploy staging (5 min). Sequential stages. Total: 45 phút per pipeline. 20 merges/day = 15 giờ pipeline time/day. Developer wait 45 min cho feedback. Productivity killed.

Optimization: DAG dependencies (parallel where possible). Build: 10 min. Parallel: lint + unit tests + integration tests = 15 min (longest). Deploy: 5 min. Total: 30 min. Further: cache dependencies, skip unchanged tests, parallel test splitting. Final: 12 min.

💡 Bài học: Sequential stages = wasted time. DAG (needs keyword) cho true dependency graph. Parallel test execution. Cache dependencies aggressively. Skip unchanged components. Pipeline time is developer time — optimize relentlessly.

💾 Chuyện Cache Miss Rebuild Everything

Node.js project, cache key: package-lock.json hash. Developer update 1 dependency → package-lock.json hash change → cache miss → npm install 3000 packages (8 phút). 15 merges/day × 8 min = 2 giờ/day npm install.

Optimization: fallback cache key (CI_DEFAULT_BRANCH), so partial cache hit even khi lock file changes. Cache trên shared runner storage. Pre-populated cache. Result: npm install 30 giây (install delta only).

💡 Bài học: Cache key strategy matters. Fallback keys cho partial cache hits. cache:key:files cho file-based keys. cache:policy: pull cho read-only jobs. Shared cache between branches via fallback. Pre-warm cache cho common dependencies.

🔐 Chuyện CI Variable Exposed

CI/CD variable DATABASE_PASSWORD set as 'Visible' (not masked, not protected). Any developer can see variable value in pipeline logs. Any branch can access variable. Fork can access variable. Credentials exposed to entire team + potential external contributors.

Fix: set variables as Protected (only protected branches) + Masked (hidden in logs). Environment-scoped variables cho production credentials. Limit access to Settings → CI/CD. Rotate exposed credentials immediately.

💡 Bài học: CI/CD variables: set Protected + Masked cho secrets. Environment-scoped cho deployment credentials. File type variables cho certificate/key files. NEVER log variable values. Audit variable access permissions. Rotate credentials khi exposed.

⚠️ Những Sai Lầm Kinh Điển

1. Monolithic .gitlab-ci.yml: 500+ lines, unmaintainable. Split with includes, templates, components. Modular CI configuration.
2. Shell executor cho shared runners: No isolation between jobs. Docker executor cho security. Shell executor only cho dedicated, single-project runners.
3. No retry strategy: Flaky tests fail pipeline. retry: 2 cho known flaky jobs. But fix flaky tests — retry is band-aid.
4. Artifacts without expiration: Artifacts stored indefinitely. Storage costs grow. Set expire_in: 1 week hoặc appropriate duration.
5. No pipeline timeout: Job stuck in loop. Default: 1 hour per job. Set appropriate timeouts. timeout: 30 minutes cho most builds.
