📜 Bí Kíp Vận Hành

1

LangChain vs LlamaIndex vs Raw API

LangChain: general-purpose framework, chains, agents, tools. Tốt cho complex multi-step workflows. Nhưng: over-abstracted, breaking changes thường xuyên. LlamaIndex: data framework, tốt nhất cho RAG + data indexing. Cleaner abstractions. Raw API: maximum control, minimum overhead. Tốt cho simple use cases. Rule: RAG → LlamaIndex. Complex agent workflows → LangChain/LangGraph. Simple prompt → raw API. Never use framework "just because."

2

Observability — Nhìn thấy bên trong LLM app

LangSmith: tracing, evaluation, playground. By LangChain team. LangFuse: open-source alternative, self-hostable. Arize Phoenix: LLM observability + traditional ML monitoring. What to trace: input/output mỗi step, latency per step, token usage, errors. Essential metrics: (1) Cost per request, (2) Latency P50/P99, (3) Error rate, (4) Quality scores (automated + human). Tracing là MUST HAVE cho production LLM apps.

3

Cost Optimization — Tiền là máu

Strategy: (1) Model routing: classify query complexity → simple → cheap model, complex → expensive model. (2) Caching: exact match cache + semantic cache (similar queries → cached response). (3) Prompt optimization: shorter prompts = fewer tokens. (4) Output limiting: max_tokens strict. (5) Batch processing: batch non-urgent requests. Monitor: cost per conversation, cost per user, daily cost trends. Set budget alerts.

4

Chain Design Patterns

Sequential Chain: A → B → C. Simple, debuggable. Router Chain: classify intent → route to specialized chain. Map-Reduce: process chunks in parallel → combine results. Good for summarization. Refine: iteratively refine answer with each chunk. Better quality nhưng chậm. LangGraph: state machine for complex flows, cycles, conditional branching. Năm 2026: LangGraph thay thế LangChain chains cho complex workflows.

5

Evaluation — Đo lường quality at scale

LLM-as-judge: dùng GPT-4/Claude đánh giá output của model khác. Cheaper than human eval, better than heuristic. Rubric-based: define clear criteria (relevance, accuracy, helpfulness, safety) trên 1-5 scale. Reference-based: compare against golden answers. A/B testing: deploy 2 versions, compare user engagement. Eval frequency: daily automated runs + weekly human review. Build eval set 100-200 representative queries.

📖 Chuyện Kể Đêm Khuya

🕸️ Chuyện 1: LangChain chain quá sâu để debug

Build customer support bot: RAG retrieval → context compression → prompt formatting → LLM call → output parsing → action routing → tool execution → response formatting. 8 steps. Bug: output sai. STEP NÀO SAI?

Không có tracing, anh Chuỗi Vận Hành phải debug bằng print statement ở mỗi step. 4 giờ tìm ra: context compression bỏ mất thông tin quan trọng ở step 2. Hôm sau: setup LangSmith tracing. Mọi step visible. Debug time: 15 phút.

💡 Bài học: Tracing/observability là KHÔNG TÙY CHỌN cho production LLM apps. Setup LangSmith/LangFuse TRƯỚC khi build complex chains. Each step should be independently testable. And: simpler chains = easier debug. 3 steps > 8 steps nếu output tương đương.
💸 Chuyện 2: Token costs bùng nổ không kiểm soát

RAG chatbot cho internal knowledge base. System prompt: 2,000 tokens. Retrieved context: 3,000 tokens. User history: 2,000 tokens. Total input per request: 7,000 tokens. 10,000 requests/ngày × 7,000 tokens × $3/1M tokens = $210/ngày = $6,300/month. Chỉ cho input tokens.

Optimize: (1) Compress system prompt 2,000 → 800 tokens, (2) Retrieve 3 chunks thay vì 5 (3,000 → 1,500), (3) Limit conversation history 5 turns (2,000 → 1,000). Total: 3,300 tokens. Cost giảm 53%. Thêm semantic cache: cost giảm thêm 30%.

💡 Bài học: LLM costs = (system prompt + context + history + user input) × requests × price per token. Mỗi component cần optimize. System prompt: ngắn gọn. Context: retrieve relevant only. History: limit turns. Cache: reduce duplicate calls. Monitor daily.
🔄 Chuyện 3: LangChain breaking changes — tuần nào cũng upgrade

Anh pin LangChain v0.0.180. 3 tháng sau, cần feature mới (LangGraph). Upgrade lên v0.1.0: 47 import errors, 12 deprecated functions, 3 API changes. Mất 2 ngày fix. Team: "Tại sao không dùng raw API từ đầu?"

Từ đó anh có rule: abstract LangChain behind custom interfaces. Nếu cần switch framework, chỉ cần rewrite interface implementation, không phải toàn bộ codebase.

💡 Bài học: LLM frameworks evolve nhanh → breaking changes thường xuyên. Protect bằng: (1) Pin versions, (2) Abstract framework behind interfaces, (3) Don't use framework-specific features unless necessary, (4) Evaluate: liệu raw API đủ cho use case? Frameworks tiết kiệm thời gian ban đầu nhưng có thể tốn thời gian dài hạn.

⚠️ Những Sai Lầm Kinh Điển

❌ Không có tracing/observability: LLM app black box = undebuggable. Setup tracing từ ngày 1.
❌ Không monitor costs: LLM costs sneak up. Daily monitoring + budget alerts + cost per user tracking.
❌ Over-engineer chains: 10-step chain khi 3-step đủ. Simpler = more reliable + easier to debug.
❌ Không cache: Same questions asked repeatedly → same expensive LLM calls. Cache saves 30-50% costs.
❌ Tightly coupled với framework: LangChain everywhere → locked in. Abstract behind interfaces.
