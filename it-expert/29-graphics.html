OpenGL Vulkan DirectX Metal Shaders GPU Pipeline

📜 Thân Thế & Lai Lịch

Trần Đồ Họa bắt đầu với OpenGL 1.0 fixed-function pipeline — khi mà 'programming GPU' có nghĩa là gọi glBegin(GL_TRIANGLES). Rồi programmable shaders ra đời, và thế giới thay đổi hoàn toàn. Ông viết vertex shader đầu tiên bằng assembly-like ARB instructions — 50 dòng để rotate một triangle. Khi Vulkan ra đời (2016), ông mất 2000 dòng code chỉ để render một tam giác — nhưng hiểu sâu hơn bao giờ hết rendering pipeline hoạt động thế nào. DirectX 12, Metal — mỗi API cho ông hiểu thêm về GPU architecture. Motto: 'GPU là máy tính song song mạnh nhất mà hầu hết developer không biết cách dùng. Hiểu GPU pipeline = unlock khả năng render hàng triệu polygons ở 60fps.'

🗝️ Bí Kíp

Bí Kíp #1: Rendering Pipeline — Từ Vertices đến Pixels

Vertex stage: transform vertices (model → world → view → clip space), vertex shaders chạy PER VERTEX. Rasterization: convert triangles → fragments (potential pixels). Fragment stage: fragment shaders chạy PER FRAGMENT — color, lighting, texturing. Output: framebuffer → screen. Modern additions: Tessellation (subdivide geometry dynamically), Geometry shader (generate/destroy primitives), Compute shader (general-purpose GPU compute). Bottlenecks: vertex-bound (too many vertices), fill-rate bound (too many pixels/overdraw), bandwidth-bound (too many texture reads).

Bí Kíp #2: Vulkan vs OpenGL — Control vs Convenience

OpenGL: state machine model, driver handles memory/synchronization, easier API, cross-platform. Nhược: driver overhead, unpredictable performance, deprecated by Apple (Metal only). Vulkan: explicit control — you manage command buffers, synchronization, memory allocation. Ưu: predictable performance, lower CPU overhead, multi-threaded rendering. Nhược: 2000+ lines cho triangle, complex API. Recommendation: OpenGL/OpenGL ES cho learning, prototyping, simple apps. Vulkan cho AAA games, professional renderers, performance-critical. WebGPU: next-gen web graphics API (Vulkan-inspired, cross-platform).

Bí Kíp #3: Shader Programming

GLSL (OpenGL), HLSL (DirectX), MSL (Metal), WGSL (WebGPU). Vertex shader: transform position, pass data to fragment. Fragment shader: calculate final pixel color (lighting, texturing, effects). Uniform variables: CPU → GPU data (matrices, time, parameters). Varying/in/out: vertex → fragment interpolated data. Performance: minimize texture samples, avoid branching (GPU SIMD prefers uniform execution), use swizzling, precompute on CPU when possible.

Bí Kíp #4: GPU Optimization

Draw calls: mỗi draw call có CPU overhead. Batch: combine meshes, instancing cho repeated objects, indirect rendering. Overdraw: pixel drawn multiple times (transparent objects, overlapping geometry). Front-to-back rendering, early-Z rejection, stencil masking. Texture bandwidth: mipmaps (automatic LOD), compressed formats (BC/ASTC), texture atlasing. Shader complexity: LOD-based shader (simpler shader for distant objects), compute work offloading.

Bí Kíp #5: Modern Rendering Techniques

PBR (Physically Based Rendering): energy-conserving materials, metallic/roughness workflow, IBL (Image Based Lighting). Deferred rendering: render geometry info to G-buffer, apply lighting in screen space — efficient cho many lights. Ray tracing: realistic reflections, shadows, GI — hardware RT cores (RTX). Nanite (Unreal): virtualized geometry, unlimited polygon count. Global Illumination: baked (lightmaps), real-time (SSGI, RTGI, Lumen). TAA/DLSS/FSR: temporal anti-aliasing, AI upscaling.

🌙 Chuyện Kể Đêm Khuya

📖 Chuyện 1: Shader Compilation Stutter — First Frame Freeze

Game release: first time player enters new area → 500ms freeze. Then smooth. Next new area → 200ms freeze. Cause: shader compilation at runtime. GPU needs compiled shader cho each unique material/lighting combination. First encounter = compile = stutter. Players review: 'Stuttering mess, unplayable.' Fix attempts: 1) Pre-warm shaders (force compile during loading screen), 2) Shader cache (compile once, cache, reuse), 3) Pipeline State Objects (PSO) pre-creation. Vulkan/DirectX 12: PSO caching. Unity: shader variant collection + warm-up. Unreal: PSO recording + replay. After fix: smooth 60fps throughout. Lesson: shader compilation stuttering = death for fast-paced games.

💡 Bài học: Shader compilation stutter là P0 issue cho gameplay experience. Solutions: 1) Pre-warm shaders during loading, 2) PSO caching (save compiled shaders to disk), 3) Reduce shader variants (feature flags), 4) Background compilation (async). Test: fresh install, first playthrough — that's what users experience. Profile cold start specifically.

📖 Chuyện 2: Z-Fighting Artifacts — Flickering Walls

Architecture visualization app: walls flickering with zebra stripe patterns. Cause: Z-fighting — two surfaces at nearly identical depth, depth buffer cannot resolve which is in front. Floating-point precision limitation: far from camera → depth buffer precision decreases. Worse with large scenes: camera can see objects from 0.1m to 10,000m → depth buffer spread thin over huge range → precision terrible at far distances. Near plane 0.001 → even worse (most precision wasted on near range). Fix: 1) Increase near clip plane (0.001 → 0.1), 2) Logarithmic depth buffer (more precision at far distances), 3) Offset coplanar surfaces slightly (glPolygonOffset), 4) Reversed depth buffer (better precision distribution). Near plane = single most important depth buffer parameter.

💡 Bài học: Z-fighting = depth buffer precision problem. Prevention: 1) Near clip as FAR as possible (0.1, not 0.001), 2) Far clip as NEAR as possible, 3) Avoid coplanar geometry (offset by small amount), 4) Reversed-Z depth buffer (float precision distribution matches 1/z better). Logarithmic depth cho massive scenes (space games, earth-scale).

📖 Chuyện 3: GPU Memory Leak — VRAM Eating Monster

Game allocates GPU textures but forgets to release them when level unloads. After 3 level transitions: VRAM full → driver starts paging to system RAM → frame drops from 60fps to 15fps → eventually crash (out-of-memory). Debug with RenderDoc/Nsight: texture count increasing with each level load. Root cause: texture references held by material instances not properly cleaned. Fix: explicit resource lifecycle management, weak references for cached textures, VRAM budget monitoring. Graphics API observation: OpenGL = driver manages memory (somewhat forgiving). Vulkan = YOU manage memory (explicit alloc/free — leak = your fault). Moving to Vulkan exposed memory management bugs that OpenGL's driver was hiding.

💡 Bài học: GPU memory management is CRITICAL. Unlike CPU memory, GPU memory leaks cause immediate visual degradation (frame drops, crashes). Profile VRAM usage: RenderDoc, GPU vendor tools (Nsight, Radeon GPU Profiler). Track allocations: texture count, buffer sizes, framebuffer attachments. Vulkan VMA (Vulkan Memory Allocator) helps manage GPU memory correctly.

💀 Sai Lầm Kinh Điển

❌ Sai Lầm #1: Overdraw — drawing hidden pixels

Transparent objects rendered back-to-front = every pixel drawn multiple times. Opaque objects: render front-to-back (early-Z rejects hidden fragments). Transparent: minimize, use alpha test (discard) when possible instead of alpha blend.

❌ Sai Lầm #2: Too many draw calls

CPU prepares each draw call: state changes, buffer binds, shader switches. 10,000 draw calls = bottleneck on CPU side. Solutions: batching, instancing, indirect rendering, merge small meshes.

❌ Sai Lầm #3: Uncompressed textures

4K uncompressed RGBA texture = 64MB VRAM. Compressed (BC7/ASTC) = 16MB. Mipmapped = even better for cache. ALWAYS use texture compression cho runtime textures. Tools: texconv, compressonator, basisu.

❌ Sai Lầm #4: Branching trong shader

GPU executes SIMD — branching causes warp/wavefront divergence. Both branches EXECUTE for ALL threads in warp. Avoid per-pixel branching. Use math tricks: step(), mix(), smoothstep() instead of if/else.

❌ Sai Lầm #5: Không profile GPU separately

CPU profiler shows CPU time. GPU runs async. Use GPU profiler (RenderDoc, Nsight, Xcode GPU Frame Capture) to see actual GPU bottlenecks: vertex processing, fragment processing, bandwidth.
