📚 Thân Thế & Lai Lịch

Đại Sư nhớ thời IPSec là lựa chọn duy nhất — 47 parameters để configure, 3 ngày để setup site-to-site tunnel, và mỗi lần debug phải đọc 500 dòng log. "Phase 1 negotiation failed" — câu này ám ảnh chàng ta suốt 10 năm.

Rồi OpenVPN xuất hiện — đơn giản hơn, UDP hoặc TCP, certificate-based. Chàng ta setup OpenVPN cho công ty: 200 remote users, split tunneling, MFA. Cuộc sống đẹp hơn.

Nhưng khi WireGuard ra đời, chàng ta phải lòng: 4000 dòng code (vs OpenVPN 100K dòng), modern cryptography, built vào Linux kernel, setup trong 5 phút. "Đây mới là VPN," chàng ta tuyên bố. Giờ chàng ta evangelizes WireGuard cho mọi người — từ enterprise đến homelab.

📜 Bí Kíp

1. WireGuard — Simple & Fast: Built vào Linux kernel (5.6+). Modern cryptography: Curve25519, ChaCha20, Poly1305. Config đơn giản: peer public key + allowed IPs. Performance: throughput gần line-rate, latency thấp. Roaming: seamless khi đổi network. Ideal cho: site-to-site, remote access, homelab.
2. OpenVPN — Battle-Tested: Mature, feature-rich, mọi platform. UDP hoặc TCP (TCP cho restrictive firewalls). Certificate-based authentication + MFA. Plugin ecosystem. Access Server cho management GUI. Nhược: slower than WireGuard, complex config. Dùng khi: cần compatibility, existing deployment, TCP tunneling.
3. IPSec/IKEv2 — Enterprise Standard: IKEv2 cho mobile devices (seamless roaming). StrongSwan cho Linux IPSec. Tích hợp tốt với cloud VPN gateways (AWS VPN, Azure VPN Gateway). Complex nhưng well-understood by enterprise firewalls. Dùng khi: enterprise requirements, cloud VPN connectivity, Windows built-in VPN client.
4. Split Tunneling: Full tunnel: ALL traffic qua VPN. Split tunnel: chỉ corporate traffic qua VPN, internet traffic direct. Trade-off: split tunnel = better performance, nhưng security risk nếu user machine compromised. Inverse split tunnel: default route qua VPN, specific IPs direct. Choose based on security requirements.
5. SD-WAN & SASE: SD-WAN: overlay network qua multiple WAN links (MPLS, broadband, LTE). Intelligent path selection. SASE (Secure Access Service Edge): SD-WAN + security (FWaaS, CASB, ZTNA) as cloud service. Vendors: Zscaler, Cloudflare, Palo Alto Prisma. Enterprise trend: replace traditional VPN with ZTNA (Zero Trust Network Access).
6. VPN Best Practices: MFA bắt buộc cho VPN access. Certificate-based auth > password auth. Log all VPN connections. Monitor concurrent connections. Bandwidth planning cho remote workforce. Kill switch cho client (đảm bảo traffic không leak ngoài VPN). Regular key rotation.

📖 Chuyện Kể Đêm Khuya

📶 Chuyện VPN MTU Issues

Remote users complain: 'Một số websites không load được qua VPN!' Debug: VPN tunnel MTU 1500, nhưng VPN overhead 80 bytes → effective MTU 1420. Large packets fragmented → fragmentation blocked bởi some firewalls → connection hangs. Websites with large responses (images, downloads) affected.

Chỉ phát hiện khi tcpdump trên VPN server thấy ICMP 'Fragmentation needed' messages bị dropped. Fix: set VPN interface MTU 1420. MSS clamping: iptables -t mangle -A FORWARD -p tcp --tcp-flags SYN,RST SYN -j TCPMSS --clamp-mss-to-pmtu.

💡 Bài học: VPN tunnel giảm effective MTU. Set interface MTU = underlying MTU - VPN overhead. MSS clamping cho TCP traffic. Test large file downloads qua VPN. MTU issues âm thầm gây problems — hard to diagnose nếu không biết look for.

🔓 Chuyện Split Tunnel Security Risk

Company cho phép split tunneling để reduce VPN bandwidth. Employee's laptop infected with malware. Malware communicate qua direct internet (không qua VPN) → bypass corporate firewall, IDS, DLP. Exfiltrate data undetected 3 tháng.

Discovery: SIEM detect unusual data patterns từ employee's account, nhưng traffic không đi qua corporate network nên logs incomplete. Investigation mất 2 tuần vì thiếu network visibility.

💡 Bài học: Split tunneling = reduced security visibility. Nếu dùng split tunnel: endpoint protection (EDR) mandatory, DNS filtering (DoH cho corporate DNS), SASE cho cloud-delivered security regardless of network path. Full tunnel cho high-security environments.

🌍 Chuyện WireGuard Homelab VPN Trong 5 Phút

Đại Sư challenge: setup VPN cho homelab access từ anywhere trong 5 phút. WireGuard server: wg genkey, config file 10 dòng. Client: scan QR code. Connected. Latency thêm 2ms. Throughput: 800Mbps trên Gigabit connection. Total time: 4 phút 30 giây.

So sánh: OpenVPN setup cùng scenario: generate CA, server cert, client cert, config file, tls-auth key — 45 phút. IPSec: 2 giờ nếu may mắn. WireGuard simplicity là game-changer cho homelab và small teams.

💡 Bài học: WireGuard tuyệt vời cho simple VPN needs. Performance gần native. Config đơn giản = ít lỗi. Nhưng: thiếu advanced features (LDAP auth, fine-grained ACLs). Cho enterprise: WireGuard + management layer (Tailscale, Netmaker, Headscale).

⚠️ Những Sai Lầm Kinh Điển

1. VPN credentials shared between employees: 1 account dùng chung = no accountability, no audit trail. Individual accounts + MFA cho mỗi user.
2. No kill switch: VPN disconnect → traffic đi direct qua internet → expose private data. Client-side kill switch block non-VPN traffic.
3. VPN server exposed directly: VPN server public IP scannable. DDoS target. Put behind DDoS protection. Use non-standard ports. Port knocking cho extra layer.
4. Không monitor VPN usage: Ai connected? Bao lâu? Bandwidth bao nhiêu? Unusual login times? VPN logs essential cho security monitoring.
5. PSK (Pre-Shared Key) cho nhiều users: 1 PSK compromise = tất cả users compromised. Certificate-based authentication hoặc WireGuard per-peer keys.
