🗝️ Bí Kíp

Bí Kíp #1: Unity vs Unreal vs Godot — The Holy Trinity

Unity: C#, massive asset store, mobile-friendly, large community. Best cho: mobile games, indie games, AR/VR, 2D games. Runtime fee controversy (đã rollback) nhưng trust damaged. Unreal: C++/Blueprints, AAA visual quality, Nanite/Lumen/MetaHuman. Best cho: AAA games, realistic graphics, large-scale open world. Free until $1M revenue. Godot: GDScript/C#/C++, open source (MIT), lightweight, node-based architecture. Best cho: indie games, 2D games, learning, prototyping. Growing community. Choose based on: target platform, visual quality needs, team expertise, budget.

Bí Kíp #2: ECS Architecture — Data-Oriented Design

Traditional OOP: GameObject with components (MonoBehaviour in Unity). ECS (Entity Component System): Entity (ID), Component (data only), System (logic operating on components). Ưu: cache-friendly (contiguous memory), parallelizable, composition over inheritance. Unity DOTS: ECS + Jobs System + Burst Compiler = massive performance. Unreal: Mass Entity Framework. Godot: node-based (not strict ECS). Khi dùng ECS: large number of entities (1000+ enemies, particles, projectiles). CRUD game UI: OOP fine.

Bí Kíp #3: Game Loop — Heartbeat của Engine

Fixed timestep cho physics (50-60Hz, deterministic), variable timestep cho rendering (as fast as possible). Unity: FixedUpdate() cho physics (fixed 0.02s), Update() cho game logic (every frame), LateUpdate() cho camera (after all Updates). Unreal: Tick function, dedicated physics sub-stepping. Godot: _physics_process(delta) fixed, _process(delta) variable. Delta time: LUÔN multiply movement by deltaTime — đảm bảo consistent speed across framerates.

Bí Kíp #4: Asset Pipeline — Import, Compress, Load

Textures: compress (ASTC cho mobile, BC7 cho desktop), mipmap chains, atlas packing cho 2D. Models: LOD (Level of Detail) — far = low poly, near = high poly. Audio: compressed formats (OGG/AAC cho music, WAV cho short SFX). Asset bundles/Addressables (Unity): load assets on demand, reduce initial load time. Unreal: Pak files, streaming. Asset naming convention: T_Character_Idle_01 (type_category_name_variant). Version control: Git LFS cho binary assets.

Bí Kíp #5: Optimization Principles

Profile BEFORE optimize (Unity Profiler, Unreal Insights, Godot profiler). GPU bound: reduce draw calls (batching, instancing, LODs), optimize shaders, reduce overdraw. CPU bound: reduce Update() logic, object pooling (don't Instantiate/Destroy), spatial partitioning (quadtree, octree). Memory: texture compression, audio compression, asset streaming. Mobile: target 30fps minimum, watch thermal throttling, reduce shader complexity, use baked lighting.

🌙 Chuyện Kể Đêm Khuya

📖 Chuyện 1: Unity Asset Store Dependency Hell

Game project dùng 15 Asset Store packages: UI framework, dialogue system, inventory, pathfinding, shader pack, 3D models, audio manager. Mỗi package works individually. Together? Namespace conflicts, duplicate utility classes, incompatible Unity versions. Unity upgrade 2021 → 2022: 5 packages chưa update → compile errors. Package A depends on Package B version 1.x, Package C depends on Package B version 2.x → conflict. Developer của Package D đã quit, source code không public → stuck với buggy version forever. Team mất 1 tháng resolve conflicts, fork abandoned packages, write adapters. Game logic chỉ chiếm 40% codebase — 60% là glue code giữa packages.

💡 Bài học: Asset Store packages = external dependencies = risk. Rules: 1) Minimize third-party packages (mỗi package = maintenance burden), 2) Prefer source code over compiled DLLs (can fix bugs), 3) Wrap packages in your own interface (swap-able), 4) Check package update frequency before adopting (abandoned = liability), 5) Build core systems in-house, use packages cho utilities.

📖 Chuyện 2: Unreal Engine Compile Times — Coffee Break Engine

Unreal C++ project: full rebuild = 45 minutes. Change one header file → 15 minutes recompile (header include chain). Developer make change → compile → go make coffee → come back → compiler error → fix typo → compile again → more coffee. Team productivity: 3-4 iterations/hour. With Unity C# (hot reload): 20+ iterations/hour. Project behind schedule. Solution: 1) Forward declarations thay vì #include, 2) PIMPL pattern reduce header dependencies, 3) Precompiled headers (PCH), 4) UnrealBuildTool: IncrediBuild for distributed compilation. Plus: use Blueprints cho rapid prototyping, convert to C++ sau khi logic finalized. Compile time giảm từ 45 phút → 8 phút. Vẫn lâu hơn Unity, nhưng livable.

💡 Bài học: C++ compile times = real productivity killer. Mitigation: 1) Minimize header includes, 2) Forward declarations, 3) PCH, 4) Distributed build systems, 5) Blueprint/visual scripting cho iteration. When choosing Unreal: factor compile times into timeline estimates. Team size matters — larger team = more conflicts = more recompilation.

📖 Chuyện 3: Physics Glitch Viral — Character Fly to Space

Third-person game: character touches corner of two colliders at exact angle → physics engine push character UP with enormous force → character flies through skybox → falls forever. Bug rare (1/1000 corner touches) nhưng player recorded video → posted Reddit → 50,000 upvotes → bug becomes FEATURE. Community names it 'Space Launch Glitch'. Speedrunners use it to skip sections. Developer dilemma: fix bug → speedrun community angry. Keep bug → new players confused. Solution: fix for normal gameplay but keep accessible via specific input sequence (deliberate vs accidental).

💡 Bài học: Physics glitches are INEVITABLE — collision detection has edge cases. Mitigate: 1) Use margin/skin width on colliders, 2) Limit max velocity, 3) Continuous collision detection (CCD) for fast objects, 4) Test corner cases literally (character vs wall corners, edge-to-edge). But also: sometimes bugs become beloved features (Rocket jumping in Quake, BLJ in Mario 64).

💀 Sai Lầm Kinh Điển

❌ Sai Lầm #1: Instantiate/Destroy mỗi frame

Creating/destroying objects = expensive (GC pressure in Unity, allocation overhead). Object pooling: pre-create, reuse, deactivate instead of destroy. Bullets, particles, enemies — pool everything that spawns frequently.

❌ Sai Lầm #2: Đặt logic trong Update() mà không cần mỗi frame

HP bar text update mỗi frame (60 times/second) → 59 lần không cần. Event-driven: update khi HP actually changes. Hoặc: coroutine update mỗi 0.5s cho non-critical UI.

❌ Sai Lầm #3: Không dùng LOD cho 3D models

10,000 poly character render ở background (5 pixels trên screen). LOD system: switch to low-poly model khi far away. Massive performance gain, minimal visual impact.

❌ Sai Lầm #4: Development build thay production build

Debug/development build: profiling overhead, debug symbols, unoptimized shaders. LUÔN test performance với PRODUCTION build. Development build có thể 2-5x slower.

❌ Sai Lầm #5: Không backup project

Unity/Unreal project file corruption = tất cả mất. Git + LFS cho version control. Regular backups. Meta files trong Unity: PHẢI commit — thiếu meta file = asset references broken.
