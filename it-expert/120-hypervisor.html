📚 Thân Thế & Lai Lịch

Ảo Hóa Chưởng Môn bắt đầu với bare metal servers — mỗi application 1 server vật lý. Server utilization: 10%. 90% wasted. "Tại sao mua Ferrari để đi 10km/h?" chàng ta hỏi.

VMware ESXi thay đổi game: 1 physical server chạy 10 VMs. Utilization 70%. Cost giảm 80%. "Virtualization is magic," chàng ta tuyên bố. Cho đến khi VMware thay đổi licensing model — per-core pricing, subscription-only, và giá tăng 3-5 lần.

Chàng ta tìm đến Proxmox VE — open source, KVM + LXC, free (với paid support optional). "VMware features at zero cost?" Community lớn, enterprise-ready, và quan trọng nhất: không bị vendor lock-in licensing surprises. Giờ chàng ta chạy cụm Proxmox 5 node cho homelab và tư vấn enterprise migration từ VMware.

📜 Bí Kíp

1. Proxmox VE — Open Source Champion: KVM cho VMs + LXC cho containers. Web UI quản lý cluster. Ceph integration cho distributed storage. ZFS support native. HA cluster với corosync. Backup built-in (PBS - Proxmox Backup Server). Free, open source, enterprise support available. Perfect cho homelab VÀ production.
2. VMware vSphere — Enterprise Standard: ESXi hypervisor + vCenter management. vMotion cho live migration. DRS cho automatic load balancing. HA cho automatic VM restart. vSAN cho software-defined storage. NSX cho networking. Powerful nhưng: licensing costs tăng significantly từ 2024. Consider alternatives nếu renewal coming.
3. KVM/QEMU — Linux Native: Built vào Linux kernel. QEMU cho device emulation. Libvirt cho management API. Virt-manager cho GUI. Cockpit cho web UI. PCI passthrough cho GPU, NVMe. Excellent performance (near-native). Used by cloud providers (AWS Nitro based on KVM). Free, no licensing concerns.
4. Live Migration: Move running VM between hosts without downtime. Requirements: shared storage hoặc storage migration, network connectivity, compatible CPU features. Use cases: hardware maintenance, load balancing, failure avoidance. Test migration TRƯỚC maintenance window. Verify network và storage connectivity.
5. Storage Strategies: Local storage: fast nhưng no shared access. NFS/iSCSI: shared storage cho migration. Ceph: distributed, self-healing, scalable. ZFS: checksums, snapshots, compression. Storage tiering: SSD cho VMs, HDD cho backups. RAID cho redundancy. ALWAYS backup regardless of storage redundancy.
6. Resource Management: CPU overcommit: safe up to 4:1 cho mixed workloads. Memory: overcommit risky — ballooning can cause issues. NUMA awareness cho large VMs. Resource pools cho team allocation. Monitoring: CPU steal, memory balloon, disk I/O latency. Capacity planning: leave 20-30% headroom.

📖 Chuyện Kể Đêm Khuya

💰 Chuyện VMware Licensing Change

Enterprise chạy VMware vSphere 200 hosts. License renewal coming. New Broadcom pricing: per-core subscription model, mandatory bundles, 3-5x price increase. Annual cost: $200K → $800K. CTO: 'WHAT?!'

Đại Sư evaluate alternatives: Proxmox VE (free, KVM-based), Nutanix AHV (included with Nutanix), bare-metal Kubernetes. Migration plan: 6 months, start with non-critical workloads, validate performance. Year 1 savings: $600K. Migration cost: $150K (staff time + training). ROI: positive within 6 months.

💡 Bài học: Vendor lock-in has real cost implications. VMware → Proxmox migration is well-documented and many enterprises doing it post-Broadcom acquisition. Plan exit strategy for ANY vendor-dependent technology. Open-source alternatives often equivalent for most workloads. Calculate TRUE TCO including licensing changes.

⚡ Chuyện Proxmox Cluster Quorum Lost

Proxmox cluster 3 nodes. Node 3 network failure. Cluster lose quorum (2/3 nodes UP but can't reach each other due to network partition). Result: 2 remaining nodes ALSO stop accepting changes — quorum requires majority. All VMs running but no management operations possible. New VMs cannot start. HA cannot failover.

Root cause: single switch connecting all 3 nodes — switch failure = total cluster management loss. Fix: redundant cluster networking (dual NICs, dual switches). Corosync: dual-ring configuration. Minimum cluster size: 3 nodes. But network redundancy MORE important than node count.

💡 Bài học: Cluster quorum critical for HA. 3-node minimum, but network is often SPOF. Dual-ring corosync networking. Redundant switches. Quorum device (QDevice) cho 2-node clusters. Test quorum loss scenarios. Understand: VMs keep running, management stops — know what to do.

🔄 Chuyện Live Migration Cứu Hardware Failure

Proxmox cluster nhận SMART warning: disk trên node 2 predicted failure trong 48 giờ. Admin login, initiate live migration cho 15 VMs từ node 2 sang remaining nodes. Migration hoàn tất trong 20 phút. Zero downtime cho users. Node 2 shutdown, disk replaced, rebuild. VMs migrated back.

Without live migration: shutdown VMs, replace disk, restart. 15 VMs × 5 minutes = 75 minutes downtime. With live migration: 0 downtime. SMART monitoring + live migration = proactive hardware management.

💡 Bài học: SMART monitoring cho predictive hardware failure detection. Live migration enables zero-downtime hardware maintenance. Shared storage hoặc storage migration required. Regular maintenance windows less critical khi live migration available. Monitor hardware health proactively.

⚠️ Những Sai Lầm Kinh Điển

1. Single host, no cluster: Host failure = all VMs down. Minimum 2 nodes cho HA, 3 nodes cho quorum. Cluster even for small setups.
2. No backup: VM snapshots ≠ backup. Regular backups to separate storage. Test restore. Proxmox Backup Server cho efficient backups.
3. Over-commit memory: Allocate 128GB RAM across VMs on 64GB host. Ballooning kicks in. VMs slow down. Memory overcommit cautiously.
4. Ignore CPU compatibility: Different CPU generations in cluster → live migration fails. EFI/CPU flags compatibility. Common CPU model cho mixed clusters.
5. No monitoring: Host runs out of storage at 2am. VM performance degraded. Monitor hypervisor resources: CPU, RAM, storage, network. Alert before problems.
