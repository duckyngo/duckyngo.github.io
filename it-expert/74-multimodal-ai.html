📜 Bí Kíp Đa Phương Thức

1

Vision-Language Models — CLIP và beyond

CLIP (OpenAI): align image + text embeddings. Zero-shot classification: encode image + encode text descriptions → cosine similarity. SigLIP: improved CLIP training. GPT-4V/4o: native multimodal, reason about images. Claude 3: strong vision understanding. LLaVA: open-source vision-language model. Use cases: image search by text, visual QA, image captioning, document understanding.

2

Multimodal Fusion Strategies

Early fusion: combine raw inputs trước processing. Simple nhưng high-dimensional. Late fusion: process mỗi modality riêng → combine at decision level. Easy to implement, modular. Cross-modal attention: modalities attend to each other (image attends to text, text attends to image). Most powerful, standard trong modern VLMs. Năm 2026: cross-attention trong transformer architecture là dominant approach.

3

Practical Multimodal Applications

(1) Document AI: image + text understanding cho invoices, contracts. (2) Visual search: "tìm sản phẩm giống ảnh này" — CLIP embeddings + vector search. (3) Video understanding: frame + audio + captions. (4) Accessibility: image → alt text, audio → transcript. (5) Medical: X-ray image + clinical notes → diagnosis. Tip: bắt đầu với API (GPT-4V, Claude Vision) → evaluate → fine-tune open model nếu cần.

4

Evaluation cho Multimodal

Khó hơn unimodal vì cần evaluate cross-modal understanding. Benchmarks: MMMU (multi-discipline), MM-Bench (perception + reasoning), VQAv2 (visual QA). Practical: build custom eval set cho your use case. Test: (1) Does model understand image content? (2) Does model connect image to text correctly? (3) Does model reason across modalities? Human eval vẫn gold standard cho multimodal quality.

5

Multimodal RAG — Beyond text

Traditional RAG: text → text. Multimodal RAG: text + images + tables → answer. Approach: (1) Extract text + images from documents. (2) Embed both (CLIP for images, text embedder for text). (3) Retrieve relevant chunks (text + images). (4) Send to multimodal LLM. Challenging: image-text alignment trong retrieval, table understanding, diagram interpretation. Tools: Unstructured for document parsing, ColPali for visual document retrieval.

📖 Chuyện Kể Đêm Khuya

🖼️ Chuyện 1: Model describes image completely wrong

GPT-4V cho product catalog: gửi ảnh áo dài đỏ. Model: "A beautiful blue dress with floral patterns." Sai màu, sai pattern, sai loại trang phục. Nhưng description nghe rất confident.

Test 100 ảnh: 85% đúng, 15% sai. Trong 15% sai: sai color (nhiều nhất), sai quantity, sai spatial relationship. Hallucination trong vision không khác text — model confident khi sai.

💡 Bài học: Multimodal models hallucinate về visual content giống text models hallucinate facts. Luôn verify critical visual information (colors, quantities, text in images). Dùng structured output + validation. Không trust model 100% cho e-commerce (sai color = wrong product shipped).
📎 Chuyện 2: CLIP search magic moment

E-commerce site: text search "áo cho mùa hè mát mẻ" → keyword search returns áo có chữ "mùa hè" trong title. CLIP search: return ảnh áo cotton nhẹ, màu pastel, sleeveless — semantically "mùa hè mát mẻ" dù title không mention.

Conversion rate tăng 23%. Users tìm được sản phẩm muốn nhanh hơn. Anh Đa Năng: "CLIP hiểu intent, keyword search hiểu words."

💡 Bài học: CLIP embeddings cho semantic visual search — users search by concept, không chỉ keywords. Combine CLIP search + traditional keyword search cho best results. Pre-compute image embeddings, search = cosine similarity trên embeddings. Fast và effective.
📊 Chuyện 3: Text-image alignment fail

Multimodal RAG cho technical documentation: diagram + text. Retrieved diagram đúng nhưng text chunk từ section khác. Model: answer dựa trên WRONG text giải thích cho RIGHT diagram. Kết quả: technically coherent nhưng factually wrong.

Fix: link diagrams to surrounding text paragraphs tại indexing time. Retrieve diagram + associated text as unit. Kết quả cải thiện 40%.

💡 Bài học: Multimodal RAG cần preserve relationships between modalities. Image detached from context = potential misinterpretation. Index image + surrounding text together. Metadata linking (figure number → paragraph reference) critical cho accuracy.

⚠️ Những Sai Lầm Kinh Điển

❌ Tin VLM description mù quáng: Models hallucinate visual details. Verify colors, quantities, text in images.
❌ Gửi ảnh resolution quá cao: 4K image → nhiều tokens → expensive + slow. Resize phù hợp (usually 768-1024px enough).
❌ Ignore image-text alignment trong RAG: Retrieved image + unrelated text = wrong answer. Keep image-text pairs together.
❌ Dùng CLIP cho fine-grained tasks: CLIP tốt cho coarse-grained (cat vs dog), kém cho fine-grained (species of cat). Fine-tune cho specific tasks.
❌ Multimodal khi unimodal đủ: Text-only task mà gửi kèm image = extra cost, no benefit. Use multimodal when multiple modalities needed.
