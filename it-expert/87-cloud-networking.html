📚 Thân Thế & Lai Lịch

Đại Sư bắt đầu với networking vật lý — kéo cáp, cấu hình switch, debug VLAN tagging. Khi chuyển sang cloud networking, chàng ta tưởng sẽ dễ hơn vì "không cần kéo cáp." Chàng ta đã sai — cloud networking phức tạp hơn, chỉ là phức tạp theo cách khác.

Thay vì debug cable connections, chàng ta debug Security Groups, NACLs, Route Tables, NAT Gateways, VPC Peering, Transit Gateway, PrivateLink, và hàng tá networking constructs mà cloud providers phát minh ra. "Tôi nhớ cái thời chỉ cần ping để biết mạng có chạy không," chàng ta thở dài.

Nhưng cloud networking cũng có magic: tạo VPC trong 10 giây thay vì 2 tuần order switch, global load balancing với DNS, và private connectivity giữa VPCs mà không cần physical cabling. Đại Sư đã master cả hai thế giới.

📜 Bí Kíp

1. VPC Design — Plan Trước Khi Tạo: CIDR block phải đủ lớn (ít nhất /16). ĐỪNG dùng 10.0.0.0/16 — conflict với mọi default VPC. Plan subnet layout: public subnets (ALB, NAT Gateway), private subnets (application), isolated subnets (database). Multi-AZ bắt buộc. Reserve CIDR space cho tương lai — VPC CIDR không thể shrink.
2. Security Groups vs NACLs: Security Groups: stateful, allow rules only, applied to ENI. NACLs: stateless, allow + deny rules, applied to subnet. Best practice: Security Groups cho fine-grained control per resource, NACLs cho subnet-level blanket rules. KHÔNG dùng 0.0.0.0/0 cho inbound rules trừ ALB/NLB public-facing.
3. Load Balancer Selection: ALB (Application LB): HTTP/HTTPS, path-based routing, host-based routing. NLB (Network LB): TCP/UDP, ultra-low latency, static IP. GLB (Gateway LB): 3rd-party appliances. ALB cho web apps, NLB cho game servers/real-time, GLB cho firewalls. Sticky sessions cẩn thận — breaks auto-scaling benefit.
4. DNS — Route 53 Best Practices: Alias records cho AWS resources (free, no CNAME limitations). Health checks + failover routing cho DR. Latency-based routing cho multi-region. Weighted routing cho canary deployments. Private hosted zones cho internal DNS. DNS TTL: short (60s) cho dynamic, long (3600s) cho stable records.
5. Transit Gateway — Hub for VPCs: Kết nối multiple VPCs + on-premise qua single gateway. Centralized routing. Shared services VPC architecture. Inter-region peering. Tốn tiền per-attachment + per-GB — plan carefully. Alternative: VPC Peering cho simple 1-to-1 connections.
6. API Gateway — Front Door cho APIs: Throttling, authentication, caching, request/response transformation. REST API vs HTTP API (cheaper, simpler). WebSocket API cho real-time. Custom domains + certificates. Usage plans + API keys cho rate limiting per customer. WAF integration cho security.

📖 Chuyện Kể Đêm Khuya

🛡️ Chuyện Security Group Allow 0.0.0.0/0

Developer tạo EC2 instance cho testing. Security group: SSH (22) from 0.0.0.0/0, all traffic outbound. 'Chỉ để test thôi mà.' Instance có public IP. 15 phút sau: SSH brute force attacks từ khắp thế giới. 2 giờ sau: instance bị compromise, bắt đầu gửi spam emails. AWS gửi abuse notification. Team scramble để contain — terminate instance, audit security groups, enable GuardDuty.

Phát hiện thêm: 12 security groups khác cũng allow 0.0.0.0/0 cho SSH. Đại Sư implement: AWS Config rule auto-detect open security groups, SSM Session Manager thay thế SSH (không cần open port), và mandatory security group review trong PR process.

💡 Bài học: KHÔNG BAO GIỜ open SSH/RDP to 0.0.0.0/0. Dùng SSM Session Manager cho shell access (không cần public IP, không cần open port). Bastion hosts nếu phải SSH — restricted to office IP. AWS Config + auto-remediation cho security group compliance.

🌍 Chuyện DNS TTL Cache 24 Giờ

Team migrate service sang new IP. Update DNS record, TTL 86400 (24 giờ). 'DNS sẽ propagate nhanh thôi.' Old server shutdown. 50% users vẫn truy cập old IP → connection timeout. Support tickets flood in. Team panic: 'DNS chưa update!' Thực tế: DNS đã update, nhưng resolvers cache record 24 giờ. Phải đợi 24 giờ cho toàn bộ users chuyển sang new IP.

Bài học đắt giá: trước migration, giảm TTL xuống 60 giây VÀI NGÀY TRƯỚC. Migrate. Sau khi confirm stable, tăng TTL lại. Simple nhưng 90% teams quên.

💡 Bài học: TTL thấp (60-300s) cho records sẽ thay đổi. Giảm TTL TRƯỚC migration, không phải lúc migration. Maintain old endpoint song song trong 24-48h. DNS propagation không instant — plan for it.

💸 Chuyện NAT Gateway Tốn $500/Tháng

Team deploy ứng dụng trong private subnets (đúng best practice). NAT Gateway cho internet access. Ứng dụng pull Docker images, npm packages, call external APIs. NAT Gateway data processing: $0.045/GB. Monthly data through NAT: 10TB. Chi phí NAT Gateway: $450/tháng data + $32/tháng hourly = $482/tháng cho... NAT.

Đại Sư optimize: VPC endpoints cho S3 và ECR (free data transfer). VPC endpoint cho DynamoDB. Caching Docker images nội bộ. Kết quả: NAT traffic giảm 80%, chi phí NAT giảm xuống $100/tháng.

💡 Bài học: NAT Gateway là hidden cost killer. VPC endpoints (Gateway endpoints cho S3/DynamoDB free, Interface endpoints cho other services ~$7/tháng) giảm NAT traffic đáng kể. Monitor NAT Gateway data processing. Consider NAT instances cho dev environments (cheaper but less managed).

⚠️ Những Sai Lầm Kinh Điển

1. VPC CIDR quá nhỏ: Tạo VPC /24 rồi hết IP khi scale. Plan for growth: /16 cho production VPC. Không thể expand CIDR sau khi tạo (chỉ thêm secondary CIDR).
2. Dùng default VPC cho production: Default VPC có public subnet by default. Mọi instance có public IP. Tạo custom VPC với private subnets cho production.
3. Overlapping CIDRs: VPC A: 10.0.0.0/16. VPC B: 10.0.0.0/16. Peering? Không thể. Plan CIDR ranges centrally. Document IP allocation.
4. Health check không đúng: Load balancer health check ping / nhưng app đã crash. Health check endpoint nên verify actual application health — database connection, dependencies.
5. Không dùng PrivateLink cho services: Access AWS services qua internet thay vì VPC endpoints. Tốn NAT Gateway costs + latency. PrivateLink keeps traffic trong AWS network.
6. DNS records không managed as code: Manual DNS changes → inconsistency, mistakes. Manage DNS records qua Terraform/CloudFormation. Version controlled, auditable.
