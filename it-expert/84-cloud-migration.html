📚 Thân Thế & Lai Lịch

Tổng Quản bắt đầu sự nghiệp trong data center, nơi chàng ta chịu trách nhiệm chuyển 500 servers từ building cũ sang building mới. Mất 6 tháng, 3 incidents mất data, và 1 lần gãy lưng (đúng nghĩa đen — server rack nặng lắm).

Khi cloud migration trở thành trend, chàng ta nghĩ: "Chuyển lên cloud chắc dễ hơn chuyển servers vật lý." Chàng ta đã sai. Cloud migration phức tạp hơn gấp bội vì phải deal với dependencies ẩn, data consistency, compliance requirements, và quan trọng nhất — sếp muốn hoàn thành trong 3 tháng cho workload 2 năm.

Sau khi cứu hỏa cho 20+ cloud migration projects, Tổng Quản nhận ra: vấn đề lớn nhất không phải kỹ thuật — nó là expectations management, skill gaps, và thiếu FinOps discipline. Giờ chàng ta dành 50% thời gian cho migration strategy và 50% cho "tại sao bill lại cao thế?"

📜 Bí Kíp

1. 7Rs Migration Strategy: Retire (decommission), Retain (keep on-premise), Rehost (lift & shift), Relocate (hypervisor-level migration), Repurchase (move to SaaS), Replatform (lift & reshape), Refactor (re-architect). Đánh giá mỗi application theo 7Rs. Không phải app nào cũng nên lên cloud.
2. Assessment Trước Khi Migration: Discovery: inventory tất cả apps, dependencies, data flows. Categorize theo business criticality, complexity, cloud readiness. TCO analysis: on-premise cost vs cloud cost (bao gồm staff, licensing, networking). Business case phải solid trước khi bắt đầu.
3. FinOps — Cloud Financial Management: FinOps = Finance + DevOps. Three phases: Inform (visibility), Optimize (right-sizing, reservations), Operate (continuous improvement). FinOps team includes finance, engineering, product. Showback/chargeback to teams. Cloud cost is everyone's responsibility.
4. Reserved Instances & Savings Plans: 1-year commitment: ~30% savings. 3-year commitment: ~60% savings. All Upfront cheapest, No Upfront most flexible. Savings Plans more flexible than RIs. NHƯNG: only commit sau khi right-size — don't reserve wrong instance types.
5. Right-Sizing — Continuous Process: Analyze CPU, memory, network, disk usage. Most instances over-provisioned 40-60%. Downsize iteratively: m5.xlarge → m5.large → t3.large. ARM instances (Graviton) 20-40% better price-performance. Right-sizing is ongoing, not one-time.
6. Database Migration — Hardest Part: Schema migration, data migration, application changes. Minimize downtime: DMS (Database Migration Service) cho continuous replication. Test extensively before cutover. Rollback plan bắt buộc. Don't change database engine AND migrate simultaneously.
7. Wave Planning — Migrate In Waves: Wave 0: Landing zone setup, networking, identity. Wave 1: Low-risk, non-critical apps. Wave 2: Medium-complexity apps. Wave 3: Complex, business-critical apps. Learn from each wave. Never start with production-critical applications.

📖 Chuyện Kể Đêm Khuya

💸 Chuyện Lift-and-Shift Đắt Gấp 10 Lần

Một enterprise lift-and-shift 50 servers lên EC2. On-premise cost: $15K/tháng (amortized hardware + power + staff). AWS cost dự kiến: $12K/tháng. Actual AWS cost: $55K/tháng.

Tại sao? On-premise servers chạy 20% CPU nhưng chia sẻ hardware. Trên cloud, mỗi VM cần dedicated instance. Storage IOPS on-premise: SAN shared, flat cost. Cloud: EBS gp3 IOPS tốn thêm. Data transfer: on-premise free giữa servers, cloud: NAT Gateway + cross-AZ charges. Licensing: SQL Server on-premise BYOL, cloud: per-vCPU licensing gấp 4x.

💡 Bài học: Lift-and-shift thường đắt hơn on-premise vì cloud pricing model khác biệt. Right-size instances TRƯỚC khi migrate. Replatform (change to managed services) tiết kiệm hơn rehost. TCO analysis phải include hidden costs: licensing, data transfer, monitoring.

💰 Chuyện FinOps Tiết Kiệm $3 Triệu/Năm

Một công ty tech đang trả $500K/tháng cho AWS. CEO hỏi: "Có thể giảm không?" FinOps team được thành lập — 2 engineers + 1 finance person. Trong 6 tháng:

Tháng 1-2: Visibility — tag resources, identify owners, create dashboards. Phát hiện $50K/tháng resources không ai nhận. Tháng 3-4: Right-sizing — 200 instances downsized, 50 instances terminated. Savings: $80K/tháng. Tháng 5-6: Commitments — Savings Plans cho stable workloads, Spot cho batch. Savings: $120K/tháng. Total: $250K/tháng savings = $3M/năm.

💡 Bài học: FinOps ROI cực cao. Đầu tư 2-3 người cho FinOps có thể save millions. Start với visibility (bạn không thể optimize cái bạn không measure), rồi optimize, rồi commit. FinOps là practice liên tục, không phải project 1 lần.

🔄 Chuyện Database Migration 72 Giờ

Migrate Oracle database 5TB từ on-premise sang RDS PostgreSQL. Plan: DMS replication 1 tuần, cutover weekend. Thực tế: DMS replication lag liên tục vì stored procedures và triggers phức tạp. Schema conversion 60% automated, 40% manual rewrite — đó là 2000 stored procedures.

Cutover weekend: application switched to new database. 15 phút sau: errors everywhere. Stored procedures không compatible. Data types khác nhau. Timezone handling different. Rollback to Oracle. Mất 3 tháng rewrite stored procedures, extensive testing, và second cutover attempt thành công.

💡 Bài học: Database engine migration (Oracle → PostgreSQL) là DỰ ÁN RIÊNG, không phải một step trong cloud migration. Schema Conversion Tool chỉ là starting point. Test EVERY stored procedure, function, trigger. Parallel run cả hai databases trước cutover. Có rollback plan solid.

⚠️ Những Sai Lầm Kinh Điển

1. Estimate migration timeline quá lạc quan: Rule of thumb: multiply ban đầu estimate bằng 3. Dependencies, testing, training, compliance — luôn mất lâu hơn nghĩ.
2. Migrate mà không right-size: Copy paste VM specs từ on-premise sang cloud = overpay. Right-size dựa trên actual usage, không phải allocated capacity.
3. Quên data transfer costs: Cross-region replication, NAT Gateway, VPN traffic, CloudFront origin fetch — tất cả tốn tiền. Data transfer có thể chiếm 20-30% total bill.
4. Commit Reserved Instances quá sớm: Mua 3-year RI trước khi right-size = lock vào wrong instance type. Right-size trước, stabilize, rồi mới commit.
5. Không có rollback plan: Mỗi migration step cần rollback procedure tested. "Chúng ta sẽ không cần rollback" — famous last words.
6. Bỏ qua licensing: Windows Server, SQL Server, Oracle licensing trên cloud phức tạp hơn on-premise. BYOL vs included licensing. vCPU-based licensing. Compliance audit from vendor. Consult licensing expert TRƯỚC migration.
7. "Big bang" migration: Migrate tất cả cùng lúc. Một lỗi ảnh hưởng toàn bộ. Wave approach an toàn hơn — learn từ mỗi wave, improve process.
