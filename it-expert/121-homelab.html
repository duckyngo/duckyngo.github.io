📜 Bí Kíp

1. Hardware Selection: Mini PCs (Intel NUC, Dell Optiplex Micro): low power (15-35W), quiet, affordable. Used enterprise servers (Dell R720, HP DL380): powerful, cheap, LOUD, power-hungry. Raspberry Pi: $35-80, perfect cho lightweight services. Build philosophy: start small (Pi + old PC), grow as needed. Power consumption matters: 200W server × 24/7 × $0.12/kWh = $210/năm.
2. ZFS — Data Integrity: Copy-on-write filesystem. Checksums detect silent data corruption (bit rot). Snapshots: instant, space-efficient. Compression: LZ4 (fast), ZSTD (better ratio). RAID levels: mirror (RAID 1), RAIDZ1 (RAID 5), RAIDZ2 (RAID 6). ECC RAM recommended (not required but helps). ARC cache in RAM. Pool degraded? Replace drive ASAP. NEVER use RAIDZ1 with drives > 4TB.
3. Self-Hosted Services: Media: Jellyfin/Plex, *arr stack (Sonarr, Radarr, Prowlarr). Productivity: Nextcloud, Paperless-ngx, Bookstack. Smart Home: Home Assistant, Node-RED. Network: Pi-hole/AdGuard, WireGuard VPN. Monitoring: Grafana + Prometheus. Reverse proxy: Traefik/Nginx Proxy Manager. Docker Compose cho deployment.
4. Backup Strategy 3-2-1: 3 copies: original + 2 backups. 2 different media: SSD + HDD, or HDD + cloud. 1 offsite: cloud backup (Backblaze B2, rclone to GCS/S3). Test restore regularly. ZFS snapshots cho quick recovery. Proxmox Backup Server cho VM backups. Duplicati/Restic cho file backups.
5. Networking: VLAN separation: management, servers, IoT, guests. Firewall rules between VLANs. DNS: Pi-hole/AdGuard Home cho ad blocking. Reverse proxy: SSL termination, domain routing. WireGuard/Tailscale cho remote access. 10Gbps between storage and compute nếu budget allows.
6. Power & Cooling: UPS bắt buộc: protect against power outages + power surges. APC, CyberPower popular cho homelab. NUT (Network UPS Tools) cho graceful shutdown. Cooling: rack fans, room ventilation. Monitor temps: IPMI, lm-sensors. Heat output substantial cho rack servers — plan accordingly.

📖 Chuyện Kể Đêm Khuya

🔋 Chuyện ZFS Pool Degraded Lúc 2 Giờ Sáng

ZFS monitoring alert: pool status DEGRADED. Drive failed. RAIDZ1 pool với 4 × 4TB drives. One drive dead. Data still accessible nhưng NO redundancy — another drive failure = total data loss. 12TB data: family photos 15 năm, documents, media library.

2am Amazon order: replacement drive, next-day delivery. 24 giờ anxiety: mỗi click sound từ remaining drives = heart attack. Drive arrives, resilver takes 18 giờ. Pool healthy again. Immediately order cold spare drive. Setup Backblaze B2 cloud backup. Never again.

💡 Bài học: RAIDZ1 với large drives = risky (resilver time long, URE during resilver possible). RAIDZ2 cho drives > 4TB. Cold spare drives ready. Offsite backup BEFORE you need it. ZFS monitoring alerts essential. 3-2-1 backup: ZFS snapshots + local backup + cloud backup.

⚡ Chuyện Power Outage Kills Array

Power outage midnight. No UPS. Rack server, NAS, networking — all hard shutdown. Power returns 2 giờ later. NAS boots: 1 drive không spin up. BTRFS array degraded. Another drive: filesystem errors. Data recovery: partially successful. Some files corrupted.

Đại Sư buy UPS same week: APC 1500VA. NUT configured: low battery → graceful shutdown all VMs → shutdown NAS → shutdown hosts. Power outage next month: UPS kicks in, 15 phút battery, graceful shutdown at 5 phút remaining. Zero data loss.

💡 Bài học: UPS is NOT optional cho homelab với important data. Hard power loss = filesystem corruption risk, drive failure risk. APC/CyberPower 1500VA sufficient cho most homelabs. NUT cho automated graceful shutdown. Test UPS regularly (NUT: upsmon). Battery replacement every 3-5 years.

🐳 Chuyện Self-Hosted Stack

Đại Sư self-host: Nextcloud (Google Drive replacement), Bitwarden/Vaultwarden (password manager), Immich (Google Photos replacement), Paperless-ngx (document management), Jellyfin (Netflix replacement). Total cloud cost replaced: $50/tháng. Homelab power cost: $30/tháng. Net savings: $20/tháng.

But real value: DATA OWNERSHIP. No Google scanning photos. No cloud provider locking account. No subscription price increases. Full control. 'My data, my rules,' Đại Sư tuyên bố.

💡 Bài học: Self-hosting saves money AND provides privacy/control. BUT requires maintenance time và knowledge. Start with easy services: Pi-hole, Nextcloud. Docker Compose cho deployment consistency. Backup CRITICAL cho self-hosted — you are the SRE now. Community support (r/selfhosted, r/homelab) invaluable.

⚠️ Những Sai Lầm Kinh Điển

1. No backup: Self-hosted data without backup. Drive dies = data gone forever. 3-2-1 backup rule. Cloud backup cho offsite.
2. No UPS: Power outage = hard shutdown = data corruption risk. UPS + NUT = graceful shutdown. Non-negotiable.
3. Enterprise server as first homelab: Dell R720: 500W idle, jet engine noise. Start with mini PCs or Raspberry Pi. Enterprise servers for dedicated space only.
4. Exposing services to internet: Port forwarding without firewall, VPN, or reverse proxy. Use Cloudflare Tunnel, WireGuard, Tailscale for safe remote access.
5. RAID without backup: RAID ≠ backup. RAID protects against drive failure. Backup protects against: user error, ransomware, fire, theft. Both needed.
