📜 Bí Kíp Phân Đoạn

1

Semantic vs Instance vs Panoptic

Semantic: mỗi pixel thuộc 1 class (sky, road, building). Không phân biệt instances. Instance: detect + segment từng object riêng biệt (car_1, car_2, car_3). Panoptic: combine cả hai — stuff (sky, road) + things (car_1, car_2). Chọn: counting objects → instance. Scene understanding → semantic. Full scene parsing → panoptic.

2

SAM — Segment Anything Model

SAM (Meta): zero-shot segmentation, prompt bằng point/box/text. SAM 2: extends to video. Use cases: (1) Automated annotation (thay vì vẽ mask tay), (2) Interactive segmentation (click to segment), (3) Foundation cho downstream tasks. FastSAM: 50x nhanh hơn SAM với CNN backbone. EfficientSAM: lightweight cho edge. Trick: SAM + YOLO = detect bounding box → SAM segment → pixel-perfect masks.

3

OCR Pipeline — Từ ảnh ra chữ

Pipeline: (1) Text Detection: tìm vùng có chữ (CRAFT, DBNet). (2) Text Recognition: đọc chữ trong vùng (CRNN, TrOCR). (3) Post-processing: spell check, format. Tools: PaddleOCR (best all-around, multilingual), EasyOCR (easy to use), Tesseract (legacy but free), TrOCR (transformer-based, high accuracy). Tiếng Việt: PaddleOCR + custom training cho fonts tiếng Việt.

4

Mask R-CNN — Instance Segmentation classic

Faster R-CNN + mask branch. Still relevant cho custom instance segmentation tasks. Detectron2 (Meta) là implementation tốt nhất. Alternatives: YOLOv8-seg (nhanh hơn, easier), SOLOv2 (no anchor, no NMS). Training tip: start with COCO pretrained, fine-tune với custom data. Loss: classification + bbox regression + mask prediction.

5

Document AI — OCR cho tài liệu cấu trúc

Beyond plain OCR: hiểu layout, tables, forms. LayoutLMv3: understand document structure + text. Donut: end-to-end document understanding without OCR. Table extraction: Camelot (PDF tables), TableTransformer (image tables). Pipeline cho hóa đơn: detect regions → OCR → NER (extract fields: tên, ngày, số tiền) → validate format.

📖 Chuyện Kể Đêm Khuya

📝 Chuyện 1: OCR đọc "1" thành "l" — và ngược lại

Hệ thống OCR đọc hóa đơn tự động. Số tiền "1,000,000 VND" bị đọc thành "l,000,000 VND" → parse error → hệ thống báo "số tiền không hợp lệ." 15% hóa đơn bị reject.

Font chữ trên hóa đơn nhiệt: "1" và "l" trông giống nhau đến 99%. Anh Pixel thêm context-aware post-processing: "nếu ở context số tiền, 'l' → '1'." Error rate giảm xuống 2%. Rồi thêm regex validation: còn 0.3%.

💡 Bài học: OCR raw output luôn cần post-processing. Context-aware correction (biết field nào expect loại data nào) cực kỳ quan trọng. Regex validation cho structured fields. Confidence threshold + human review cho low-confidence reads. OCR = 60% recognition + 40% post-processing.
🩸 Chuyện 2: Segmentation mask "chảy máu"

Segment sản phẩm trên ảnh e-commerce để tách nền. Mask nhìn OK trên ảnh nhỏ, nhưng zoom in: mask boundary "chảy" — pixels ở edge bị sai, tạo halo artifact khi paste lên nền mới.

Root cause: model predict mask ở resolution thấp (28×28 trong Mask R-CNN), upsample lên original size → boundary blurry. Fix: (1) PointRend refine edges, (2) SAM cho boundary precision, (3) Alpha matting cho hair/fur. Kết quả: clean cutout không thua Photoshop (nhiều).

💡 Bài học: Segmentation mask resolution matters cho production. Default Mask R-CNN mask (28×28) không đủ cho precise cutouts. Solutions: PointRend, CascadePSP, SAM refinement, hoặc alpha matting. Evaluate mask quality ở full resolution, không chỉ ở metric level.
🤖 Chuyện 3: SAM + YOLO = Dream Team

Cần segment 50 loại sản phẩm khác nhau trong warehouse. SAM: segment tốt nhưng không biết object là gì. YOLO: detect và classify tốt nhưng chỉ bounding box. Solution: YOLO detect → crop box → SAM segment trong box → perfect instance masks với class labels.

Kết quả: 92% mAP instance segmentation mà không cần train segmentation model riêng. Training cost: chỉ cần bbox annotations (dễ hơn mask annotations 10x).

💡 Bài học: Combine strengths: YOLO cho detection + classification, SAM cho segmentation. Annotation effort giảm đáng kể (bbox vs mask). SAM as zero-shot segmentor + task-specific detector = powerful combo cho production systems.

⚠️ Những Sai Lầm Kinh Điển

❌ OCR mà không preprocessing image: Ảnh nghiêng, nhiễu, low contrast → OCR fail. Deskew, denoise, binarize trước.
❌ Dùng Tesseract cho mọi thứ: Tesseract tốt cho printed text rõ ràng. Handwriting, curved text, scene text → dùng PaddleOCR hoặc TrOCR.
❌ Evaluate segmentation chỉ bằng IoU trung bình: High average IoU có thể mask poor performance trên specific classes. Report per-class IoU.
❌ SAM cho mọi bài toán: SAM zero-shot tốt, nhưng domain-specific trained model thường tốt hơn cho specific tasks.
❌ Mask R-CNN default mask resolution: 28×28 mask không đủ cho precise boundaries. Cần refinement cho production quality.
